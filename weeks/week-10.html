1. หลักการออกแบบ UI/UX

UI (User Interface) คือสิ่งที่ผู้ใช้มองเห็นและสัมผัส ได้แก่ สีสัน รูปทรง Typography Layout ปุ่มและองค์ประกอบต่าง ๆ ในขณะที่ UX (User Experience) คือประสบการณ์โดยรวมของผู้ใช้ว่าแอปพลิเคชันใช้งานง่าย เข้าใจง่าย มีประสิทธิภาพและน่าพอใจหรือไม่ UI ที่สวยงามแต่ UX แย่ (เช่น ปุ่มหาไม่เจอ, ขั้นตอนซับซ้อน) ก็ถือว่าล้มเหลว

หลักการออกแบบพื้นฐานที่ควรรู้: Visual Hierarchy คือการจัดลำดับความสำคัญของข้อมูลด้วยขนาด น้ำหนัก และสีเพื่อนำสายตาผู้ใช้, Color Theory ใช้ Primary, Secondary, Accent Color ให้สอดคล้องกัน และมี Contrast เพียงพอ (WCAG แนะนำอย่างน้อย 4.5:1 สำหรับ Normal Text), Typography เลือก Font ที่อ่านง่าย ขนาดเหมาะสม (Body text ควรไม่ต่ำกว่า 16px), Whitespace พื้นที่ว่างช่วยให้เนื้อหาหายใจและลดความอึดอัด

8-Point Grid System คือการออกแบบให้ทุก Spacing, Padding, Margin, Width, Height เป็นคูณของ 8 เช่น 8, 16, 24, 32, 48, 64 px ทำให้ Layout สม่ำเสมอและมีความสอดคล้องกันทั้งหน้า ง่ายต่อการพัฒนาและแก้ไข

เคล็ดลับ

ใช้ CSS Custom Properties (Variables) สำหรับสีและขนาดที่ใช้ซ้ำ เช่น --color-primary: #2563eb; และ --spacing-4: 1rem; ทำให้แก้ไข Design ได้ง่ายในจุดเดียว

2. Responsive Web Design

Responsive Web Design คือการออกแบบเว็บให้แสดงผลได้ดีบนทุก Device ตั้งแต่มือถือ (320-767px), Tablet (768-1023px) ไปจนถึง Desktop (1024px+) โดยใช้ CSS Media Queries เพื่อกำหนด Style ที่แตกต่างกันตามขนาดหน้าจอ

แนวคิด Mobile-First คือการเขียน CSS สำหรับมือถือก่อน แล้วเพิ่ม Media Query สำหรับ Tablet และ Desktop ทีหลัง วิธีนี้ดีกว่า Desktop-First เพราะมือถือมีข้อจำกัดมากกว่า (หน้าจอแคบ, เน็ตช้า) ทำให้คิดถึง Core Content ก่อน และโหลดได้เร็วกว่า เพราะ CSS พื้นฐานเบากว่า

อย่าลืม <meta name="viewport" content="width=device-width, initial-scale=1.0"> ใน <head> ของทุกหน้า หากขาด Tag นี้ มือถือจะ Scale หน้าเว็บลงให้พอดีหน้าจอ ทำให้ข้อความเล็กและอ่านยาก แม้จะมี Media Query ก็ไม่ทำงานถูกต้อง

  /* Mobile First — ไม่มี Media Query = ใช้กับมือถือ */
  .card-grid { display: grid; grid-template-columns: 1fr; }

  /* Tablet — หน้าจอ ≥ 768px */
  @media (min-width: 768px) {
    .card-grid { grid-template-columns: repeat(2, 1fr); }
  }

  /* Desktop — หน้าจอ ≥ 1024px */
  @media (min-width: 1024px) {
    .card-grid { grid-template-columns: repeat(3, 1fr); }
  }

3. CSS Grid และ Flexbox สำหรับ Layout

Flexbox เหมาะสำหรับ Layout 1 มิติ (แถวเดียวหรือคอลัมน์เดียว) เช่น Navigation Bar, Toolbar, Row ของ Card, จัด Align ตัวเองตรงกลาง Flexbox ควบคุมได้ว่าจะจัดตามแนวนอน (flex-direction: row) หรือแนวตั้ง (flex-direction: column) และใช้ justify-content, align-items จัดตำแหน่ง

CSS Grid เหมาะสำหรับ Layout 2 มิติ (แถวและคอลัมน์พร้อมกัน) เช่น หน้า Product Catalog, Dashboard Layout, Photo Gallery คุณสมบัติที่มีประโยชน์มาก: grid-template-columns: repeat(auto-fill, minmax(250px, 1fr)) จะสร้าง Column อัตโนมัติตามพื้นที่ที่มี โดยแต่ละ Column กว้างอย่างน้อย 250px ทำให้ Responsive โดยไม่ต้องเขียน Media Query เพิ่มเลย

Unit fr (Fraction) ใน CSS Grid คือการแบ่งพื้นที่ที่เหลือ เช่น grid-template-columns: 1fr 2fr 1fr แบ่งเป็น 3 Column สัดส่วน 1:2:1 fr จะคำนวณจากพื้นที่ที่เหลือหลังหัก Fixed Width และ Gap ออกแล้ว

หมายเหตุ

ในทางปฏิบัติ Flexbox และ Grid ทำงานร่วมกันได้ดีมาก ใช้ Grid สำหรับ Page Layout หลัก และใช้ Flexbox สำหรับ Component ภายใน เช่น ปุ่มหรือ Nav Item

4. Bootstrap เบื้องต้น

Bootstrap คือ CSS Framework ที่ได้รับความนิยมสูงสุดในโลก ช่วยให้สร้าง UI ที่ดูสวยงามได้รวดเร็วโดยไม่ต้องเขียน CSS เองมาก นำเข้าผ่าน CDN ด้วย <link> และ <script> ไม่กี่บรรทัด

ระบบ Grid ของ Bootstrap ใช้ Container → Row → Col เพื่อจัด Layout ซึ่งมี Breakpoint 5 ขนาด: sm (≥576px), md (≥768px), lg (≥992px), xl (≥1200px), xxl (≥1400px) เช่น col-12 col-md-6 col-lg-4 หมายความว่า มือถือเต็มแถว, Tablet ครึ่งแถว, Desktop 1 ใน 3

Utility Classes ที่ใช้บ่อย: d-flex, justify-content-between, align-items-center, text-center, mt-3 (margin-top), p-4 (padding), shadow, rounded, bg-primary text-white เป็นต้น Component พร้อมใช้ได้แก่ Navbar, Card, Button, Modal, Alert, Badge, Table

5. การออกแบบ Form ที่ดี

Form ที่ดีต้องใช้งานได้ง่าย ไม่สร้างความสับสน ปัจจัยสำคัญ: (1) Label ชัดเจน ทุกช่อง Input ต้องมี Label ที่อ่านแล้วรู้ทันทีว่าต้องกรอกอะไร ไม่ควรใช้ Placeholder แทน Label เพราะ Placeholder หายไปเมื่อเริ่มพิมพ์ (2) Error States แสดง Error ที่ใกล้ชิดกับ Input ที่ผิด ไม่ใช่ยิงรวมไว้ด้านบนทั้งหมด และ Highlight Input ที่มี Error ด้วยสีแดง (3) Success States แสดง Checkmark หรือสีเขียวเมื่อข้อมูลถูกต้อง (4) Disabled State ปุ่ม Submit ควร Disable ระหว่างกำลัง Submit เพื่อป้องกัน Double Submit

Accessibility สำคัญมาก: ใช้ for attribute บน Label เชื่อมกับ id ของ Input, ใช้ aria-label สำหรับ Input ที่ไม่มี Label แสดงผล, ใช้ aria-describedby เชื่อม Input กับข้อความ Error, ตรวจสอบว่าสามารถใช้งานได้ด้วยแป้นพิมพ์เท่านั้น (Tab, Enter, Escape)

ข้อควรระวัง

อย่าพึ่งพา Client-Side Validation เพียงอย่างเดียว (HTML5 required, pattern) ผู้ใช้สามารถ Bypass ได้ด้วย Browser DevTools เสมอต้องมี Server-Side Validation ด้วย

🧪 ปฏิบัติการ Lab 9 — UI/UX & Responsive Web

ออกแบบหน้าเว็บที่สวยงามและใช้งานได้บนทุกหน้าจอ ตั้งแต่ Product Catalog, Navigation, Dashboard ไปจนถึงการทดสอบด้วย DevTools

1
ออกแบบ Responsive Product Catalog

สร้างหน้า catalog.html ที่แสดงสินค้าอย่างน้อย 6 ชิ้นใน Card Layout โดยใช้ CSS Grid + Media Query แบบ Mobile-First: 1 Column บนมือถือ (ไม่มี Media Query), 2 Columns บน Tablet (≥640px), 3 Columns บน Desktop (≥1024px)

แต่ละ Card ต้องมี: รูปภาพสินค้า (ใช้ placeholder image ได้), ชื่อสินค้า, ราคา, และปุ่ม "เพิ่มลงตะกร้า" ที่ Hover แล้วเปลี่ยนสี Card ทั้งหมดต้องมีความสูงสม่ำเสมอ (ใช้ grid-row: span หรือ Flexbox ภายใน Card)

เกณฑ์การผ่าน

ลดขนาดหน้าต่าง Browser: Desktop เห็น 3 คอลัมน์ / 640px เห็น 2 คอลัมน์ / <640px เห็น 1 คอลัมน์ / Card มีรูป ชื่อ ราคา ปุ่ม / Hover Card มี Effect

คำแนะนำ

เริ่มจาก CSS ที่ไม่มี Media Query ก่อน (Mobile) แล้วค่อยเพิ่ม @media (min-width: 640px) ทดสอบระหว่างทำโดยลดขนาดหน้าต่าง Browser ให้แคบลง

2
Responsive Navigation Bar

สร้าง Navigation Bar ที่: Desktop แสดงเมนูแนวนอนทั้งหมด, Mobile ซ่อนเมนูและแสดงปุ่ม Hamburger (☰) เมื่อกด Hamburger ให้เมนูเด้งออกมา (Toggle)

สามารถใช้ Pure CSS Checkbox Hack หรือ JavaScript Toggle Class ก็ได้ Nav ต้องติด (sticky/fixed) อยู่บนสุดขณะ Scroll ด้วย

เกณฑ์การผ่าน

Desktop: เห็นเมนูทั้งหมด / Mobile (<768px): เห็นแค่ Logo + Hamburger / กด Hamburger → เมนูเด้งออกมา / Nav ติดบนสุดขณะ Scroll ลง

คำแนะนำ

วิธีง่ายที่สุดคือใช้ Bootstrap 5 Navbar ที่มี Collapse built-in แต่ถ้าต้องการทำเองให้ใช้ JavaScript: document.querySelector('.nav-menu').classList.toggle('open') แล้วกำหนด CSS สำหรับ class open

3
Bootstrap Dashboard Layout

สร้างหน้า dashboard_bs.html โดยใช้ Bootstrap 5 นำเข้าผ่าน CDN ออกแบบ Layout ที่มี: Navbar บนสุด, Sidebar ซ้าย (ซ่อนบน Mobile), Main Content ขวา

ใน Main Content ให้มี Stats Cards อย่างน้อย 4 ใบ (เช่น สินค้า, สมาชิก, ยอดขาย, คำสั่งซื้อ) และตาราง "สินค้าล่าสุด" อย่างน้อย 5 แถว ใช้ Bootstrap Grid (col-*), Card Component, และ Table Component

เกณฑ์การผ่าน

Desktop: เห็น Sidebar + Main Content / Mobile: Sidebar ซ่อน เห็นแต่ Main Content / Stats Cards แสดง 2 ต่อแถวบน Mobile, 4 ต่อแถวบน Desktop / ตารางมี Responsive Scroll

คำแนะนำ

ใช้ d-none d-md-block เพื่อซ่อน Sidebar บน Mobile และใช้ table-responsive ครอบตาราง เพื่อให้เลื่อนซ้าย-ขวาได้บนจอแคบ

4
ออกแบบ Login/Register Form ให้สวยงาม

นำ login.php และ register.php จาก Week 9 มาเพิ่ม CSS ให้สวยงาม ต้องมี: Card กลางหน้าที่มี Shadow และ Border Radius, Floating Labels (Label ลอยขึ้นเมื่อ Focus/มีค่า), Error State แสดงสีแดงที่ Input ที่ผิด, ปุ่ม Submit ที่ดูน่ากด

แบบฟอร์มต้องใช้งานได้ดีบนมือถือด้วย: Input ต้องกว้างเต็มหน้าจอ, ปุ่มต้องใหญ่พอแตะได้ (อย่างน้อย 44px สูง), ไม่มีการ Zoom เมื่อ Tap บน iOS (font-size อย่างน้อย 16px)

เกณฑ์การผ่าน

Form มี Shadow + Rounded Card / Label เคลื่อนเมื่อ Focus / กรอกข้อมูลผิด → Input เป็นสีแดง / ใช้งานบนมือถือได้สะดวก ไม่ต้อง Zoom / ปุ่ม Hover มี Effect

คำแนะนำ

ป้องกัน Auto-Zoom บน iOS ด้วยการใช้ font-size: 16px ขึ้นไปสำหรับ Input ทุกตัว และตั้ง touch-action: manipulation เพื่อลด Tap Delay

5
ทดสอบ Responsive ด้วย Chrome DevTools

เปิด Chrome DevTools (F12) แล้วสลับไปโหมด Device Toolbar (Ctrl+Shift+M) ทดสอบ catalog.html และ dashboard_bs.html ที่ Breakpoint ต่อไปนี้ และแก้ไขปัญหาที่พบ:

  • iPhone SE — 375 x 667 px
  • iPad Air — 820 x 1180 px
  • Desktop — 1440 x 900 px

สิ่งที่ต้องตรวจสอบ: ข้อความไม่ถูก Clip, ไม่มี Horizontal Scroll, ปุ่มมีขนาดพอสำหรับนิ้วมือ, รูปภาพไม่ล้น Container, ไม่มีพื้นที่ว่างผิดปกติ

เกณฑ์การผ่าน

ทั้ง 3 Breakpoint: ไม่มี Horizontal Scroll / ทุกเนื้อหาอ่านได้ / ปุ่มและ Link กดได้สะดวก / Layout ไม่แตก / บันทึก Screenshot แต่ละ Breakpoint ส่งอาจารย์
