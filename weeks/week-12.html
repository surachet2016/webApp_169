Web Service คืออะไร?

Web Service คือระบบซอฟต์แวร์ที่ออกแบบมาเพื่อให้เครื่องคอมพิวเตอร์หรือโปรแกรมต่างๆ สามารถสื่อสารและแลกเปลี่ยนข้อมูลกันผ่านเครือข่ายได้โดยอัตโนมัติ (Machine-to-Machine Communication) แทนที่จะเป็นมนุษย์ที่เข้าไปคลิกหน้าเว็บเอง

ในอดีต Web Service ยุคแรกใช้โปรโตคอล SOAP (Simple Object Access Protocol) ซึ่งมีความซับซ้อนสูง ใช้ XML เป็นรูปแบบข้อมูล และต้องการ WSDL เพื่ออธิบาย API ปัจจุบันโลกเปลี่ยนมาใช้ REST (Representational State Transfer) เกือบทั้งหมด เพราะง่ายกว่า เบากว่า และใช้ JSON ซึ่งอ่านและประมวลผลได้ง่ายกว่า XML มาก

ตัวอย่างการใช้ Web Service ในชีวิตจริง:

  • แอปบน iPhone เรียก API ของ Server เพื่อดึงข้อมูลโปรไฟล์ผู้ใช้
  • เว็บ E-Commerce เรียก API ธนาคารเพื่อตรวจสอบการชำระเงิน
  • แอปพยากรณ์อากาศดึงข้อมูลจาก Weather API
  • แอป Ride-sharing เรียก Google Maps API เพื่อแสดงเส้นทาง
หมายเหตุ

JSON (JavaScript Object Notation) คือรูปแบบข้อมูลที่ได้รับความนิยมสูงสุดสำหรับ REST API มีโครงสร้างคล้าย Object ใน JavaScript และ Array ใน PHP ทุกภาษาโปรแกรมมีไลบรารีสำหรับ parse JSON ทำให้เหมาะสำหรับการสื่อสารระหว่างระบบที่ต่างภาษากัน

REST API หลักการ

REST ไม่ใช่โปรโตคอล แต่เป็นชุดข้อจำกัด (Constraints) สำหรับการออกแบบ API ที่ดี หลักการสำคัญ:

  • Stateless — แต่ละ Request ต้องมีข้อมูลครบถ้วนในตัวเอง Server ไม่เก็บ "สถานะ" ของ Client ระหว่าง Request
  • Client-Server — แยก Frontend และ Backend ออกจากกันชัดเจน สื่อสารผ่าน API เท่านั้น
  • Uniform Interface — ใช้ URL มาตรฐานเพื่อระบุ Resource และ HTTP Methods เพื่อกำหนด Action
  • Resource-based — ทุกอย่างเป็น "Resource" ที่มี URL เฉพาะ เช่น /api/products, /api/users/5

HTTP Methods และความหมายใน REST:

  • GET /api/products — อ่านรายการสินค้าทั้งหมด (Read)
  • GET /api/products/5 — อ่านสินค้า ID 5 (Read One)
  • POST /api/products — สร้างสินค้าใหม่ (Create)
  • PUT /api/products/5 — อัปเดตสินค้า ID 5 ทั้งหมด (Update)
  • PATCH /api/products/5 — อัปเดตสินค้า ID 5 บางฟิลด์ (Partial Update)
  • DELETE /api/products/5 — ลบสินค้า ID 5 (Delete)

HTTP Status Codes ที่ใช้บ่อยใน API:

  • 200 OK — สำเร็จ (GET, PUT, PATCH, DELETE)
  • 201 Created — สร้างข้อมูลสำเร็จ (POST)
  • 400 Bad Request — ข้อมูลที่ส่งมาไม่ถูกต้องหรือไม่ครบ
  • 401 Unauthorized — ต้องยืนยันตัวตนก่อน (ไม่มี Token)
  • 403 Forbidden — ไม่มีสิทธิ์เข้าถึง (มี Token แต่ไม่ใช่ Admin)
  • 404 Not Found — ไม่พบ Resource ที่ขอ
  • 500 Internal Server Error — เกิดข้อผิดพลาดบน Server

สร้าง REST API ด้วย PHP

การสร้าง REST API ด้วย PHP ไม่ซับซ้อน เพียงแค่:

  1. ตั้งค่า Content-Type เป็น application/json ด้วย header()
  2. ตรวจสอบ HTTP Method ที่ส่งมาด้วย $_SERVER['REQUEST_METHOD']
  3. ประมวลผลตาม Method และส่ง JSON กลับด้วย json_encode()
  4. ตั้ง HTTP Status Code ด้วย http_response_code()

ในโปรเจกต์ขนาดเล็ก เราสามารถสร้างไฟล์ API แต่ละไฟล์ตาม Resource เช่น api/products.php, api/users.php, api/orders.php แต่ละไฟล์จะจัดการ Methods ต่างๆ (GET, POST, PUT, DELETE) สำหรับ Resource นั้น

เคล็ดลับ

เพิ่ม CORS headers ให้ API ถ้าต้องการให้ Frontend อื่น (เช่น React app บน port อื่น) เรียกได้: header('Access-Control-Allow-Origin: *'); สำหรับ Production ให้กำหนด domain ที่เฉพาะเจาะจงแทน * เพื่อความปลอดภัย

เรียกใช้ API ด้วย JavaScript fetch()

fetch() คือ Web API ของ Browser ที่ใช้สำหรับเรียก HTTP Request แบบ Asynchronous ทำให้เราโหลดข้อมูลจาก API และแสดงผลโดยไม่ต้อง Reload หน้า วิธีนี้เรียกว่า AJAX (Asynchronous JavaScript And XML) แม้ปัจจุบันจะใช้ JSON แทน XML แล้ว

fetch() ทำงานแบบ Promise-based และเราใช้ async/await ทำให้โค้ดอ่านง่ายขึ้น:

  • await fetch(url) — ส่ง HTTP Request และรอ Response
  • response.ok — true ถ้า Status Code อยู่ระหว่าง 200-299
  • await response.json() — Parse JSON จาก Response Body
  • ใช้ try/catch จัดการ Error เช่น Network Error หรือ API Error

CORS (Cross-Origin Resource Sharing) — เมื่อ JavaScript บน domain A พยายามเรียก API บน domain B Browser จะบล็อกคำขอถ้า Server B ไม่อนุญาต ต้องตั้ง Access-Control-Allow-Origin header บน Server API

เรียกใช้ API จาก PHP

บางครั้ง Backend PHP ของเราต้องเรียก API ภายนอก เช่น Weather API, Exchange Rate API, SMS Gateway PHP มีสองวิธีหลัก:

  • file_get_contents() + stream_context_create() — ง่าย เหมาะสำหรับ GET request บาง Hosting อาจปิด allow_url_fopen ทำให้ใช้ไม่ได้
  • cURL — ยืดหยุ่นกว่า รองรับทุก HTTP Method, Header, Authentication ทำงานได้กับทุก Hosting แนะนำให้ใช้เป็นหลักสำหรับ Production
ข้อควรระวัง

อย่าเก็บ API Key หรือ Secret Token ไว้ใน Code ที่ Push ขึ้น Git เพราะจะถูกเห็นโดยสาธารณะ ให้เก็บใน config.php หรือ .env file และเพิ่ม config.php ใน .gitignore API Key ที่หลุดออกไปอาจทำให้เกิดค่าใช้จ่ายหรือการละเมิดความปลอดภัย

async function loadProducts() {
    const container = document.getElementById('product-list');
    container.innerHTML = '
กำลังโหลด...
';

    try {
        const response = await fetch(API_URL);

        if (!response.ok) {
            throw new Error(`HTTP error: ${response.status}`);
        }

        const result = await response.json();

            throw new Error(result.message || 'เกิดข้อผิดพลาด');

        document.getElementById('total-count').textContent = result.total;

ไม่สามารถโหลดข้อมูลได้: ${error.message}
`;

}

async function addProduct(event) {
    event.preventDefault();

    const form = event.target;
    const data = {
        name:  form.name.value.trim(),
        price: parseFloat(form.price.value),
        stock: parseInt(form.stock.value, 10) || 0,
    };

    if (!data.name || isNaN(data.price)) {
        alert('กรุณากรอกชื่อสินค้าและราคาให้ครบถ้วน');
        return;
    }

    try {
        const response = await fetch(API_URL, {
            method:  'POST',
            headers: { 'Content-Type': 'application/json' },
            body:    JSON.stringify(data),
        });

        const result = await response.json();

        if (result.status === 'success') {
            form.reset();
            await loadProducts(); // โหลดรายการใหม่
            showMessage('เพิ่มสินค้าสำเร็จ!', 'success');
        } else {
            showMessage(result.message, 'error');
        }
    } catch (error) {
        showMessage('เกิดข้อผิดพลาดในการเชื่อมต่อ', 'error');
    }
}

function escapeHtml(str) {
    const div = document.createElement('div');
    div.appendChild(document.createTextNode(str));

}

function curlGet(string $url): array {
    $ch = curl_init();
    curl_setopt_array($ch, [
        CURLOPT_URL            => $url,
        CURLOPT_RETURNTRANSFER => true,
        CURLOPT_FOLLOWLOCATION => true,
        CURLOPT_TIMEOUT        => 10,
        CURLOPT_USERAGENT      => 'PHP Web Application',
        CURLOPT_SSL_VERIFYPEER => true,
    ]);

    $response  = curl_exec($ch);
    $httpCode  = curl_getinfo($ch, CURLINFO_HTTP_CODE);
    $curlError = curl_error($ch);
    curl_close($ch);

    if ($curlError) {
        return ['success' => false, 'data' => $curlError, 'http_code' => 0];
    }

    return [
        'success'   => ($httpCode >= 200 && $httpCode < 300),
        'data'      => json_decode($response, true),
        'http_code' => $httpCode,
    ];
}

const API = '/api/products.php';

async function api(method, body = null) {
    const options = {
        method,
        headers: { 'Content-Type': 'application/json' },
    };
    if (body) options.body = JSON.stringify(body);

    const res  = await fetch(API, options);
    const json = await res.json();

    if (!res.ok) throw new Error(json.message || `HTTP ${res.status}`);
    return json;
}

สร้าง REST API ด้วย PHP และเรียกใช้ด้วย JavaScript fetch() รวมถึงการเรียก API ภายนอก

1
สร้าง REST API พื้นฐาน

สร้างไฟล์ api/products.php ที่รองรับ 2 Methods:

  1. สร้างโฟลเดอร์ api/ ในโปรเจกต์และสร้างไฟล์ api/products.php
  2. ตั้งค่า Content-Type เป็น application/json และ CORS headers
  3. GET Request: ดึงสินค้าทั้งหมดจากตาราง products และคืน JSON ในรูปแบบ {"status":"success","data":[...],"total":N}
  4. POST Request: รับ JSON จาก Request Body (php://input), Validate ว่ามี name และ price, INSERT เข้า DB, คืน 201 พร้อม id ที่สร้างใหม่
  5. ถ้า Method ไม่รองรับ คืน 405 Method Not Allowed
เกณฑ์การผ่าน

GET /api/products.php คืน JSON array ของสินค้า, POST ที่ส่ง {"name":"ทดสอบ","price":100} สร้างสินค้าใหม่และคืน 201 พร้อม id ที่ถูกต้อง

คำแนะนำ

อ่าน Request Body ด้วย $body = json_decode(file_get_contents('php://input'), true); และตรวจสอบ Method ด้วย $_SERVER['REQUEST_METHOD']

2
ทดสอบ API ด้วย curl

ทดสอบ API ที่สร้างขึ้นด้วย curl ใน Terminal (Command Prompt / Git Bash / Terminal):

  1. เปิด XAMPP ให้ Apache และ MySQL ทำงานอยู่
  2. ทดสอบ GET: curl -i http://localhost/your-project/api/products.php
  3. ตรวจสอบว่า Response มี Header Content-Type: application/json และ Status Code 200
  4. ทดสอบ POST: curl -i -X POST -H "Content-Type: application/json" -d '{"name":"สินค้าทดสอบ","price":150,"stock":10}' http://localhost/your-project/api/products.php
  5. ตรวจสอบว่า Response มี Status Code 201 และ JSON ที่มี id
  6. ทดสอบ POST ที่ไม่มี name: ต้องได้ Status 400 พร้อม error message
เกณฑ์การผ่าน

ทดสอบได้ทั้ง 3 กรณีและได้ Status Code ถูกต้อง (200, 201, 400) พร้อม JSON Response ที่มีรูปแบบสมบูรณ์

คำแนะนำ

บน Windows ถ้าใช้ CMD ให้ใช้เครื่องหมายคำพูดคู่ (") แทน (' ) ใน JSON: -d "{\"name\":\"test\",\"price\":100}" หรือใช้ PowerShell / Git Bash ที่รองรับ single quote

3
เรียก API ด้วย JavaScript fetch()

สร้างหน้า HTML ที่โหลดข้อมูลสินค้าจาก API โดยไม่ Reload หน้า:

  1. สร้างไฟล์ products-spa.php หรือ products-spa.html
  2. เพิ่ม <div id="product-list"></div> สำหรับแสดงสินค้า
  3. เขียน JavaScript ใช้ async function loadProducts() ที่เรียก fetch('/api/products.php')
  4. แปลง Response เป็น JSON ด้วย response.json()
  5. วนลูปแสดงสินค้าทุกชิ้นใน #product-list ในรูปแบบ Card
  6. เรียก loadProducts() เมื่อ DOMContentLoaded
เกณฑ์การผ่าน

เปิดหน้า products-spa ในเบราว์เซอร์แล้วเห็นรายการสินค้าแสดงโดยไม่ต้อง Reload หน้า ตรวจสอบใน DevTools Network tab ว่ามี Request ไปยัง api/products.php

คำแนะนำ

ใช้ try/catch รอบ fetch() เสมอ และตรวจสอบ response.ok ก่อน parse JSON เพื่อจัดการกรณีที่ API คืน Error Status Code

4
สร้างฟอร์ม AJAX

เพิ่มฟอร์มเพิ่มสินค้าในหน้า products-spa ที่ส่งข้อมูลผ่าน fetch() โดยไม่ Reload หน้า:

  1. เพิ่มฟอร์ม HTML: input สำหรับ name, price, stock และปุ่ม Submit
  2. เพิ่ม Event Listener submit บนฟอร์ม และเรียก event.preventDefault()
  3. อ่านค่าจากฟอร์มและสร้าง Object JSON
  4. เรียก fetch(API_URL, { method: 'POST', body: JSON.stringify(data) })
  5. หลังเพิ่มสำเร็จ: clear form, เรียก loadProducts() ใหม่, แสดง success message
  6. ถ้า API คืน error: แสดง error message โดยไม่ Reload หน้า
เกณฑ์การผ่าน

กรอกฟอร์มและกด Submit แล้วเห็นสินค้าใหม่ปรากฏในรายการโดยไม่ Reload หน้า ตรวจสอบฐานข้อมูลว่าข้อมูลถูกบันทึกจริง

คำแนะนำ

อย่าลืมตั้ง Header 'Content-Type': 'application/json' ใน fetch options ไม่เช่นนั้น PHP จะไม่สามารถ parse php://input เป็น JSON ได้

5
เรียก API ภายนอก

ทดลองเรียก Public API ที่ไม่ต้องใช้ API Key และแสดงผลบนหน้าเว็บ:

  1. เลือก API หนึ่งในนี้ (ล้วนฟรีและไม่ต้อง Key):
    • https://restcountries.com/v3.1/name/thailand — ข้อมูลประเทศไทย
    • https://jsonplaceholder.typicode.com/posts?_limit=5 — ข้อมูล Mock posts
    • https://api.frankfurter.app/latest?from=USD&to=THB — อัตราแลกเปลี่ยน
  2. สร้างไฟล์ PHP ที่ใช้ cURL เรียก API ที่เลือก
  3. Parse JSON Response และแสดงข้อมูลที่น่าสนใจบนหน้าเว็บ
  4. จัดการ Error กรณีที่ API ภายนอกไม่ตอบสนอง (cURL error หรือ HTTP error)
เกณฑ์การผ่าน

หน้าเว็บแสดงข้อมูลจาก External API ได้สำเร็จ มีการจัดการ Error กรณี API ล้มเหลว และข้อมูลที่แสดงถูกต้องตาม Response จริง

คำแนะนำ

ตั้ง CURLOPT_TIMEOUT เสมอ (เช่น 10 วินาที) เพื่อป้องกันหน้าเว็บค้างถ้า External API ตอบช้า และตรวจสอบ curl_errno($ch) ก่อนใช้งาน Response
