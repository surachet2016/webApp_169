ภาพรวมของการพัฒนาเว็บแอปพลิเคชัน

การพัฒนาเว็บแอปพลิเคชัน (Web Application Development) คือกระบวนการสร้างโปรแกรมที่ทำงานผ่านเว็บเบราว์เซอร์ โดยผู้ใช้งานสามารถเข้าถึงได้ผ่านอินเทอร์เน็ตหรือเครือข่ายภายใน (Intranet) โดยไม่ต้องติดตั้งซอฟต์แวร์เพิ่มเติม บนเครื่องของตนเอง

ความแตกต่างระหว่าง เว็บไซต์ (Website) และ เว็บแอปพลิเคชัน (Web Application) อยู่ที่ระดับของการโต้ตอบกับผู้ใช้ เว็บไซต์ทั่วไปมักแสดงข้อมูลแบบ Static เช่น หน้าโบรชัวร์บริษัทหรือบล็อก ในขณะที่เว็บแอปพลิเคชันมีการโต้ตอบสูง ผู้ใช้สามารถป้อนข้อมูล บันทึก แก้ไข และดำเนินการต่าง ๆ ได้

ตัวอย่างเว็บแอปพลิเคชันในชีวิตประจำวัน ได้แก่:

  • E-Commerce — Shopee, Lazada, Amazon: ผู้ใช้สามารถค้นหาสินค้า เพิ่มลงตะกร้า ชำระเงิน และติดตามพัสดุ
  • Social Media — Facebook, Instagram, Twitter: ผู้ใช้โพสต์ข้อความ รูปภาพ แสดงความคิดเห็น
  • Internet Banking — KBank, SCB Online: โอนเงิน ตรวจสอบยอด จ่ายบิล
  • Learning Management System (LMS) — ระบบลงทะเบียนเรียน ดาวน์โหลดเอกสาร ส่งงาน
  • Google Docs / Gmail — แก้ไขเอกสารและส่งอีเมลผ่านเบราว์เซอร์โดยตรง
หมายเหตุ

รายวิชานี้จะสอนให้คุณสร้างเว็บแอปพลิเคชันจริงตั้งแต่พื้นฐาน โดยเริ่มจาก HTML/CSS/JavaScript ไปจนถึงการเชื่อมต่อฐานข้อมูล MySQL ด้วย PHP และการ Deploy ขึ้น Server

สถาปัตยกรรม Client-Server

การทำงานของเว็บแอปพลิเคชันพึ่งพาโมเดล Client-Server ซึ่งแบ่งการทำงานออกเป็นสองฝั่งหลัก: Client คือเบราว์เซอร์ของผู้ใช้ และ Server คือเครื่องคอมพิวเตอร์ที่เก็บโปรแกรมและข้อมูล

เมื่อผู้ใช้พิมพ์ URL ในเบราว์เซอร์ (เช่น https://www.example.com) เบราว์เซอร์จะส่ง HTTP Request ไปยัง Web Server ซึ่ง Server จะประมวลผลและส่ง HTTP Response กลับมา โดยทั่วไปจะเป็นไฟล์ HTML, CSS, JavaScript หรือข้อมูล JSON 

┌─────────────────────────┐                    ┌─────────────────────────┐
│      Client Browser     │                    │       Web Server        │
│                         │                    │                         │
│  ผู้ใช้พิมพ์ URL         │                    │  Apache / Nginx         │
│  หรือกดปุ่ม Submit       │                    │  ประมวลผล PHP           │
│                         │  ──HTTP Request──► │  เชื่อมต่อ MySQL        │
│  เบราว์เซอร์แสดงผล HTML  │  ◄──HTTP Response─ │  ส่งผลลัพธ์กลับ         │
│  รัน JavaScript         │                    │                         │
└─────────────────────────┘                    └─────────────────────────┘

ฝั่ง Client (Front-end) รับผิดชอบสิ่งที่ผู้ใช้มองเห็นและโต้ตอบ ประกอบด้วย: HTML (โครงสร้าง), CSS (รูปแบบ), JavaScript (ลอจิกฝั่ง Client) ส่วนฝั่ง Server (Back-end) รับผิดชอบการประมวลผลข้อมูล การเข้าถึงฐานข้อมูล และความปลอดภัยของระบบ

โพรโทคอล HTTP

HTTP (HyperText Transfer Protocol) คือโพรโทคอลหลักที่ใช้สื่อสารระหว่าง Client และ Server บนเว็บ โดย HTTP กำหนดรูปแบบของ Request และ Response รวมถึง HTTP Methods ต่าง ๆ ที่บ่งบอกจุดประสงค์ของการร้องขอ:

  • GET — ขอข้อมูลจาก Server (เช่น โหลดหน้าเว็บ ค้นหาข้อมูล) ข้อมูลจะแสดงใน URL
  • POST — ส่งข้อมูลไปให้ Server (เช่น ส่งฟอร์มลงทะเบียน) ข้อมูลซ่อนอยู่ใน Request Body
  • PUT / PATCH — อัปเดตข้อมูลที่มีอยู่แล้วบน Server
  • DELETE — ลบข้อมูลบน Server

HTTP Status Codes คือรหัสตัวเลขที่ Server ส่งกลับมาพร้อม Response เพื่อบอกสถานะของการร้องขอ:

  • 200 OK — สำเร็จ ปกติ
  • 201 Created — สร้างข้อมูลใหม่สำเร็จ
  • 301 / 302 — Redirect ไปยัง URL อื่น
  • 400 Bad Request — Request ไม่ถูกต้อง (ข้อมูลไม่ครบ)
  • 401 Unauthorized — ต้อง Login ก่อน
  • 403 Forbidden — ไม่มีสิทธิ์เข้าถึง
  • 404 Not Found — ไม่พบหน้าที่ร้องขอ
  • 500 Internal Server Error — เกิดข้อผิดพลาดบน Server
เคล็ดลับ

เปิด Chrome DevTools (กด F12) แล้วไปที่แท็บ Network จากนั้นโหลดหน้าเว็บใดก็ได้ คุณจะเห็น HTTP Requests และ Responses ทั้งหมดที่เกิดขึ้น รวมถึง Status Code, Headers, และ Response Body นี่เป็นเครื่องมือสำคัญที่นักพัฒนาใช้ Debug ปัญหาเว็บ

เทคโนโลยีที่ใช้ในรายวิชา

ในรายวิชานี้เราจะใช้เทคโนโลยีหลัก 4 กลุ่มที่ทำงานร่วมกันเพื่อสร้างเว็บแอปพลิเคชันครบวงจร:

  • HTML / CSS / JavaScript (Front-end) — HTML กำหนดโครงสร้างหน้าเว็บ, CSS ตกแต่งรูปลักษณ์, JavaScript เพิ่มการโต้ตอบและดึงข้อมูลแบบ Dynamic ทั้งสามทำงานบนเบราว์เซอร์ของผู้ใช้
  • PHP (Server-side Scripting) — ภาษาโปรแกรมฝั่ง Server ที่รันบน Web Server ใช้ประมวลผลข้อมูลจากฟอร์ม เชื่อมต่อฐานข้อมูล สร้างหน้า HTML แบบ Dynamic เป็นที่นิยมมากสำหรับเว็บแอปพลิเคชัน
  • MySQL (Relational Database) — ระบบจัดการฐานข้อมูลที่เก็บข้อมูลในรูปแบบตาราง ใช้คำสั่ง SQL (SELECT, INSERT, UPDATE, DELETE) ในการจัดการข้อมูล ทำงานร่วมกับ PHP ได้อย่างราบรื่น
  • XAMPP (Local Development Server) — ซอฟต์แวร์ที่รวม Apache (Web Server), MySQL, PHP, และ phpMyAdmin ไว้ในชุดเดียว ติดตั้งง่าย ใช้สำหรับพัฒนาและทดสอบเว็บบนเครื่องของตนเอง ก่อน Deploy ขึ้น Server จริง 
┌──────────────────────────────────────────────────────────────┐
│                    Web Application Stack                      │
├──────────────────────────────────────────────────────────────┤
│  Front-end (Browser)        │  Back-end (Server)             │
│  ─────────────────────────  │  ───────────────────────────── │
│  HTML  — โครงสร้าง          │  PHP    — ประมวลผล Logic       │
│  CSS   — รูปลักษณ์          │  MySQL  — จัดเก็บข้อมูล        │
│  JS    — โต้ตอบ Dynamic     │  Apache — ให้บริการ HTTP       │
└──────────────────────────────────────────────────────────────┘

เครื่องมือที่จำเป็น

ก่อนเริ่มพัฒนาเว็บแอปพลิเคชัน คุณต้องเตรียมเครื่องมือพื้นฐาน 3 อย่างดังนี้:

  • VS Code (Visual Studio Code) — Text Editor ที่ได้รับความนิยมสูงสุดในหมู่นักพัฒนา ดาวน์โหลดได้ฟรีจาก code.visualstudio.com มี Extensions มากมาย เช่น PHP Intelephense, Live Server, Prettier สำหรับช่วยพัฒนา
  • Google Chrome + DevTools — เบราว์เซอร์สำหรับทดสอบ พร้อม Developer Tools ที่ช่วย Inspect HTML/CSS, Debug JavaScript, ดู Network Requests, และตรวจสอบ Performance เปิดได้โดยกด F12 หรือ Ctrl+Shift+I (Mac: Cmd+Option+I)
  • XAMPP — ดาวน์โหลดจาก apachefriends.org ใช้สำหรับรัน PHP และ MySQL บนเครื่องของตนเอง หลังติดตั้ง ให้เปิด XAMPP Control Panel และ Start Apache กับ MySQL จากนั้นวางไฟล์ PHP ใน folder htdocs และเข้าถึงได้ที่ http://localhost/
ข้อควรระวัง

XAMPP เหมาะสำหรับการพัฒนาบนเครื่องส่วนตัวเท่านั้น ห้ามใช้บน Production Server เนื่องจากการตั้งค่าเริ่มต้น ไม่มีความปลอดภัยเพียงพอ เช่น ไม่มีรหัสผ่านสำหรับ MySQL root user

devtools-demo.js JavaScript 

// ======================================================
// การใช้งาน Chrome DevTools — Console Tab
// กด F12 แล้วคลิก "Console" เพื่อทดลองโค้ดด้านล่างนี้
// ======================================================

// 1. แสดงข้อความใน Console
console.log('สวัสดี นี่คือ Console!');
console.warn('นี่คือ Warning (สีเหลือง)');
console.error('นี่คือ Error (สีแดง)');

// 2. ดูข้อมูล Object
const student = { name: 'สมชาย', id: '6601001', gpa: 3.75 };
console.log('ข้อมูลนักศึกษา:', student);
console.table([student]); // แสดงแบบตาราง

// 3. ตรวจสอบ element ใน DOM
const heading = document.querySelector('h1');
console.log('พบ Element:', heading);
console.log('ข้อความ:', heading.textContent);

// 4. วัดเวลาประมวลผล
console.time('ทดสอบเวลา');
let sum = 0;
for (let i = 0; i < 1000000; i++) sum += i;
console.timeEnd('ทดสอบเวลา');
console.log('ผลรวม:', sum);

// ======================================================
// Network Tab — ดู HTTP Requests
// ======================================================
// วิธีใช้: เปิด Network tab ก่อน แล้วโหลดหน้าใหม่ (F5)
// คุณจะเห็น:
//   - ทุก Request ที่เบราว์เซอร์ส่ง (HTML, CSS, JS, รูปภาพ)
//   - Status Code ของแต่ละ Request (200, 404, ฯลฯ)
//   - เวลาที่ใช้ในการโหลด
//   - Headers ของ Request และ Response

// ทดลอง fetch ข้อมูลจาก API แล้วดูใน Network tab
fetch('https://jsonplaceholder.typicode.com/todos/1')
  .then(response => {
    console.log('Status:', response.status); // 200
    console.log('Headers:', response.headers);
    return response.json();
  })
  .then(data => {
    console.log('ข้อมูลที่ได้:', data);
    // { userId: 1, id: 1, title: "...", completed: false }
  })
  .catch(error => console.error('เกิดข้อผิดพลาด:', error));
