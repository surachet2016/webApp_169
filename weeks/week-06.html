1. CRUD คืออะไร?

CRUD ย่อมาจาก Create, Read, Update, Delete ซึ่งเป็นการดำเนินการพื้นฐาน 4 อย่างที่ทุกแอปพลิเคชันที่ใช้ฐานข้อมูลจำเป็นต้องมี ไม่ว่าจะเป็นระบบจัดการสินค้า ระบบสมาชิก หรือแม้แต่โซเชียลมีเดีย ล้วนสร้างขึ้นจาก CRUD ทั้งสิ้น

ในภาษา SQL แต่ละตัวอักษรของ CRUD ตรงกับคำสั่งดังนี้: Create → INSERT, Read → SELECT, Update → UPDATE, Delete → DELETE และเมื่อนำมาใช้งานในเว็บแอปพลิเคชัน PHP เราจะเชื่อมต่อผ่าน PDO (PHP Data Objects) ซึ่งเป็นวิธีที่ปลอดภัยและยืดหยุ่นที่สุดในการติดต่อกับฐานข้อมูล

ความสำคัญของการเข้าใจ CRUD คือการที่นักพัฒนาจะสามารถสร้างฟีเจอร์ใหม่ ๆ ได้อย่างรวดเร็ว เพราะทุกระบบสามารถแยกย่อยออกมาเป็น CRUD ได้เสมอ ตัวอย่างเช่น ระบบโพสต์บล็อกก็ประกอบด้วย สร้างโพสต์ (Create), แสดงโพสต์ (Read), แก้ไขโพสต์ (Update), และลบโพสต์ (Delete)

  C — Create  → INSERT INTO table (col1, col2) VALUES (?, ?)
  R — Read    → SELECT * FROM table WHERE id = ?
  U — Update  → UPDATE table SET col1 = ? WHERE id = ?
  D — Delete  → DELETE FROM table WHERE id = ?

2. Create — เพิ่มข้อมูล (INSERT)

การเพิ่มข้อมูลเริ่มต้นจากการสร้าง HTML Form ที่มี method="POST" ผู้ใช้กรอกข้อมูล แล้ว Submit ฟอร์ม PHP จะรับข้อมูลผ่าน $_POST แล้วทำการ INSERT เข้าฐานข้อมูลผ่าน PDO Prepared Statement

สิ่งสำคัญที่ต้องทำหลัง INSERT สำเร็จคือการ Redirect ผู้ใช้ไปยังหน้าอื่น เช่น หน้ารายการสินค้า นี่คือรูปแบบที่เรียกว่า Post/Redirect/Get (PRG Pattern) ซึ่งป้องกันไม่ให้ผู้ใช้กด Refresh แล้วเกิดการส่งข้อมูลซ้ำโดยไม่ตั้งใจ หากไม่ใช้ PRG Pattern เมื่อผู้ใช้กด F5 บนหน้าผลลัพธ์ เบราว์เซอร์จะถามว่า "ต้องการส่งข้อมูลอีกครั้งหรือไม่?" ซึ่งอาจทำให้ข้อมูลถูกเพิ่มซ้ำได้

ก่อน INSERT ควรทำการ Validate ข้อมูลทุกครั้ง เช่น ตรวจสอบว่าฟิลด์บังคับกรอกครบหรือไม่ ตรวจสอบรูปแบบ email, ตรวจสอบว่าตัวเลขอยู่ในช่วงที่กำหนด และ Sanitize ข้อมูลก่อนนำไปใช้เพื่อป้องกัน XSS

เคล็ดลับ — PRG Pattern

หลัง INSERT สำเร็จให้ใช้ header('Location: list.php'); exit; เสมอ เพื่อป้องกันการ Submit ซ้ำเมื่อผู้ใช้กด Refresh หน้า

3. Read — อ่านข้อมูล (SELECT)

การอ่านข้อมูลจากฐานข้อมูลสามารถทำได้หลายรูปแบบ ตั้งแต่ SELECT ข้อมูลทั้งหมด (SELECT * FROM products), SELECT แบบมีเงื่อนไข (SELECT * FROM products WHERE id = ?), ไปจนถึงการค้นหาด้วย LIKE (WHERE name LIKE ?) หรือการเรียงลำดับด้วย ORDER BY

เมื่อข้อมูลมีจำนวนมาก ควรใช้ Pagination เพื่อแบ่งหน้า โดยใช้ LIMIT และ OFFSET ใน SQL เช่น SELECT * FROM products LIMIT 10 OFFSET 20 จะดึงข้อมูล 10 รายการ โดยข้ามแถวแรกไป 20 แถว ซึ่งเทียบเท่ากับหน้า 3 เมื่อแสดงหน้าละ 10 รายการ

ข้อมูลที่ดึงมาจาก PDO จะอยู่ในรูปแบบ Array ที่สามารถวนซ้ำด้วย foreach เพื่อแสดงเป็นตาราง HTML ได้ทันที ควรใช้ htmlspecialchars() ทุกครั้งเมื่อแสดงข้อมูลใน HTML เพื่อป้องกัน XSS Attack

หมายเหตุ

เมื่อ SELECT ข้อมูลที่อาจมีอักขระพิเศษ เช่น <, >, " ให้ใช้ htmlspecialchars($value, ENT_QUOTES, 'UTF-8') ทุกครั้งก่อนแสดงผลใน HTML

4. Update — แก้ไขข้อมูล (UPDATE)

กระบวนการ Update มี 2 ขั้นตอนหลัก คือ (1) โหลดข้อมูลเดิม ด้วย SELECT มาแสดงในฟอร์มก่อน (pre-fill form) เพื่อให้ผู้ใช้เห็นข้อมูลปัจจุบันและแก้ไขเฉพาะส่วนที่ต้องการเปลี่ยน จากนั้น (2) เมื่อผู้ใช้ Submit ฟอร์ม ให้ทำ UPDATE ลงฐานข้อมูล

การส่ง ID ของรายการที่ต้องการแก้ไขทำได้ผ่าน URL Parameter เช่น edit.php?id=5 ซึ่ง PHP รับได้ผ่าน $_GET['id'] และต้องตรวจสอบว่า ID นั้นถูกต้องและมีอยู่จริงในฐานข้อมูลก่อนแสดงฟอร์ม เพื่อป้องกันผู้ใช้ที่ตั้งใจแก้ไข URL เพื่อเข้าถึงข้อมูลที่ไม่ได้รับอนุญาต

ข้อควรระวัง

อย่าลืม WHERE ใน UPDATE และ DELETE! ถ้าลืม WHERE จะแก้ไข/ลบข้อมูลทั้งหมดในตาราง เช่น UPDATE products SET price = 0 โดยไม่มี WHERE จะทำให้สินค้าทุกชิ้นราคาเป็น 0 ทันที!

5. Delete — ลบข้อมูล (DELETE)

การลบข้อมูลออกแบบ 2 แบบ คือ Hard Delete ซึ่งลบออกจากฐานข้อมูลจริง ๆ ข้อมูลหายถาวร และ Soft Delete ซึ่งเพิ่ม Column deleted_at หรือ is_deleted แล้ว UPDATE แทนการลบจริง วิธีนี้ทำให้สามารถกู้คืนข้อมูลได้ภายหลังหากลบผิดพลาด ระบบงานจริงมักใช้ Soft Delete

ก่อนลบควรแสดง Confirmation Dialog หรือหน้า Confirm เพื่อให้ผู้ใช้ยืนยันก่อน ป้องกันการลบโดยไม่ตั้งใจ ไม่ควรให้ลบด้วย GET Request เพราะ Bot หรือ Crawler อาจติดตาม Link และลบข้อมูลโดยไม่ตั้งใจ ควรลบด้วย POST เสมอ

หากข้อมูลที่จะลบมีความสัมพันธ์กับตารางอื่น (Foreign Key) ต้องตรวจสอบว่าการลบจะทำให้ข้อมูลในตารางอื่นเสียหายหรือไม่ก่อน อาจต้องลบข้อมูลในตารางที่เกี่ยวข้องก่อน (CASCADE DELETE) หรือป้องกันการลบหากยังมีข้อมูลอ้างอิงอยู่

  Hard Delete:   DELETE FROM products WHERE id = 5
                 → ข้อมูลหายถาวร ไม่สามารถกู้คืนได้

  Soft Delete:   UPDATE products SET deleted_at = NOW() WHERE id = 5
                 → ข้อมูลยังอยู่ในฐานข้อมูล สามารถกู้คืนได้
                 → SELECT ต้องเพิ่ม WHERE deleted_at IS NULL เสมอ

$errors = [];

if ($_SERVER['REQUEST_METHOD'] === 'POST') {
    $name        = trim($_POST['name'] ?? '');
    $description = trim($_POST['description'] ?? '');
    $price       = $_POST['price'] ?? '';
    $stock       = $_POST['stock'] ?? '';

    if ($name === '') $errors[] = 'กรุณากรอกชื่อสินค้า';
    if (!is_numeric($price) || $price < 0) $errors[] = 'ราคาต้องเป็นตัวเลขและไม่ติดลบ';

    if (empty($errors)) {
        $sql = "UPDATE products
                SET name = :name, description = :description,
                    price = :price, stock = :stock
                WHERE id = :id";
        $stmt = $pdo->prepare($sql);
        $stmt->execute([
            ':name'        => $name,
            ':description' => $description,
            ':price'       => (float)$price,
            ':stock'       => (int)$stock,
            ':id'          => $id,

        header('Location: list.php?success=updated');
        exit;
    }

    // อัปเดตค่าใน $product สำหรับแสดงใน form หากมี error
    $product = array_merge($product, compact('name', 'description', 'price', 'stock'));

🧪 ปฏิบัติการ Lab 5 — CRUD Operations

สร้างระบบจัดการสินค้าสมบูรณ์แบบครอบคลุมการสร้าง อ่าน แก้ไข และลบข้อมูลผ่าน PHP และ MySQL

1
สร้างตาราง products ในฐานข้อมูล

เปิด phpMyAdmin และสร้างตาราง products ในฐานข้อมูลที่สร้างไว้จากสัปดาห์ก่อน โดยมี Column ดังนี้:

สร้างตาราง products SQL 
CREATE TABLE products (
    id          INT AUTO_INCREMENT PRIMARY KEY,
    name        VARCHAR(255) NOT NULL,
    description TEXT,
    price       DECIMAL(10,2) NOT NULL DEFAULT 0.00,
    stock       INT NOT NULL DEFAULT 0,
    created_at  TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at  TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

หลังสร้างตาราง ให้ INSERT ข้อมูลตัวอย่างอย่างน้อย 3 รายการเพื่อใช้ทดสอบ

เกณฑ์การผ่าน

ตาราง products มีครบทุก Column ที่กำหนด และมีข้อมูลอย่างน้อย 3 รายการในตาราง ตรวจสอบด้วย SELECT * FROM products ใน phpMyAdmin

คำแนะนำ

ใช้ DECIMAL(10,2) สำหรับราคาเพื่อให้เก็บทศนิยมได้ถูกต้อง ไม่ควรใช้ FLOAT เพราะอาจเกิดความคลาดเคลื่อนในการคำนวณเงิน

2
หน้าเพิ่มสินค้า (create.php)

สร้างไฟล์ create.php ที่มีทั้ง HTML Form และตรรกะการบันทึกข้อมูลในไฟล์เดียวกัน ต้องมีฟิลด์: ชื่อสินค้า, คำอธิบาย, ราคา, จำนวนสต็อก เมื่อ Submit ให้ INSERT ลงฐานข้อมูลและ Redirect ไป list.php

ต้องมีการตรวจสอบความถูกต้อง (Validation) อย่างน้อย: ชื่อสินค้าต้องไม่ว่าง, ราคาและสต็อกต้องเป็นตัวเลขที่ไม่ติดลบ หากข้อมูลผิดให้แสดง Error Message และรักษาข้อมูลที่ผู้ใช้กรอกไว้ไม่ให้หาย

เกณฑ์การผ่าน

กรอกข้อมูลครบแล้วกด Submit ข้อมูลต้องปรากฏใน list.php และฐานข้อมูล / เมื่อกรอกข้อมูลผิด (เช่น ราคาเป็นตัวอักษร) ต้องแสดงข้อความ Error ที่ชัดเจน

คำแนะนำ

ใช้ is_numeric() ตรวจสอบตัวเลข และ trim() ตัดช่องว่างหน้า-หลังก่อน Validate เสมอ อย่าลืม require 'db.php' ที่บรรทัดแรก

3
หน้าแสดงรายการสินค้า (list.php)

สร้างไฟล์ list.php ที่ดึงสินค้าทั้งหมดจากฐานข้อมูลและแสดงเป็น Table ที่มี Column: ID, ชื่อสินค้า, คำอธิบาย, ราคา, สต็อก, และปุ่มจัดการ (Edit / Delete)

ปุ่ม "แก้ไข" ต้อง Link ไป edit.php?id=X และปุ่ม "ลบ" ต้อง Link ไป delete.php?id=X นอกจากนี้ให้แสดง Flash Message เมื่อกลับมาหลัง Create/Update/Delete สำเร็จ

เกณฑ์การผ่าน

หน้าแสดงสินค้าทั้งหมดในตาราง / Link ไปหน้า Edit และ Delete ทำงานถูกต้อง / Flash Message แสดงเมื่อดำเนินการสำเร็จ

คำแนะนำ

ใช้ number_format($p['price'], 2) แสดงราคาให้มีทศนิยม 2 ตำแหน่ง และใช้ htmlspecialchars() กับทุกข้อมูลที่แสดงใน HTML เพื่อความปลอดภัย

4
หน้าแก้ไขสินค้า (edit.php)

สร้างไฟล์ edit.php ที่รับ ID ผ่าน URL (?id=X) แล้ว SELECT ข้อมูลสินค้ามาแสดงใน Form (pre-fill) เมื่อ Submit ให้ UPDATE ข้อมูลในฐานข้อมูลและ Redirect กลับ list.php

ต้องตรวจสอบว่า ID ที่รับมามีอยู่ในฐานข้อมูลจริง ถ้าไม่มีให้แสดงข้อความ "ไม่พบสินค้า" Validation ต้องเหมือน create.php

เกณฑ์การผ่าน

กดปุ่ม "แก้ไข" จาก list.php → ฟอร์มแสดงข้อมูลเดิมครบถ้วน → แก้ไขและบันทึก → ข้อมูลใน list.php อัปเดตถูกต้อง / ทดสอบด้วย URL id ที่ไม่มีอยู่จริง ต้องแสดง error

คำแนะนำ

แปลง ID เป็น Integer ด้วย (int)$_GET['id'] เสมอก่อนใช้งาน เพื่อป้องกัน SQL Injection แม้จะใช้ Prepared Statement แล้วก็ตาม

5
หน้าลบสินค้า (delete.php)

สร้างไฟล์ delete.php ที่แสดงรายละเอียดสินค้าที่จะลบพร้อมปุ่ม "ยืนยันการลบ" และปุ่ม "ยกเลิก" การลบจริงต้องทำผ่าน POST Request เท่านั้น ไม่ใช่ GET

เมื่อผู้ใช้ยืนยัน ให้ DELETE ข้อมูลจากฐานข้อมูลและ Redirect กลับ list.php พร้อม Flash Message "ลบสินค้าสำเร็จ"

เกณฑ์การผ่าน

กดปุ่ม "ลบ" จาก list.php → แสดงหน้ายืนยันพร้อมข้อมูลสินค้า → กด "ยืนยัน" → ข้อมูลหายจาก list.php / กด "ยกเลิก" → กลับ list.php โดยไม่ลบ

คำแนะนำ

ให้ใช้ if ($_SERVER['REQUEST_METHOD'] === 'POST') ตรวจสอบก่อน DELETE เสมอ ป้องกันการลบผ่าน URL โดยตรง (GET Request)
