ตัวแปรและโครงสร้างควบคุม PHP

PHP มีโครงสร้างควบคุม (Control Structures) เพื่อกำหนดทิศทางการทำงานของโปรแกรมตามเงื่อนไข และทำซ้ำการทำงานบางส่วน ซึ่งเป็นพื้นฐานสำคัญของทุกโปรแกรม

เงื่อนไข (Conditional Statements):

  • if / elseif / else — ตรวจสอบเงื่อนไขตามลำดับ ใช้ === เพื่อเปรียบเทียบ ทั้งค่าและชนิดข้อมูล (Strict Comparison) แทน == ที่ทำ Type Coercion
  • switch — เปรียบเทียบค่าตัวแปรกับหลาย Cases เหมาะเมื่อมีหลายตัวเลือก อย่าลืม break; ในแต่ละ case ไม่เช่นนั้นจะ Fall-through ไป case ถัดไป
  • Ternary Operator: $result = $condition ? 'yes' : 'no'; ใช้แทน if/else แบบสั้นเมื่อมีเงื่อนไขเดียว
  • Null Coalescing Operator: $name = $_GET['name'] ?? 'ผู้เยี่ยมชม'; ส่งคืนค่าด้านซ้ายหากมีค่าและไม่ใช่ null ไม่เช่นนั้นส่งคืนค่าด้านขวา (PHP 7+)

ลูป (Loops):

  • for ($i = 0; $i < 10; $i++) — วนซ้ำที่รู้จำนวนรอบล่วงหน้า
  • foreach ($array as $item) — วนซ้ำทุก Element ใน Array (แนะนำสำหรับ Array)
  • foreach ($array as $key => $value) — วนซ้ำพร้อมดึง Key และ Value
  • while ($condition) — วนซ้ำขณะที่เงื่อนไขเป็น true
  • do { ... } while ($condition); — รันอย่างน้อยหนึ่งครั้งก่อนตรวจเงื่อนไข
เคล็ดลับ — Alternative Syntax

เมื่อผสม PHP กับ HTML ใช้ Alternative Syntax เพื่อให้อ่านง่ายขึ้น: if (...): ... elseif (...): ... else: ... endif; และ foreach (...): ... endforeach; แทนการใช้ { } แบบปกติ

ฟังก์ชันใน PHP

ฟังก์ชัน (Functions) คือบล็อกโค้ดที่ตั้งชื่อไว้เพื่อนำกลับมาใช้ซ้ำ ช่วยลดการเขียนโค้ดซ้ำ (DRY: Don't Repeat Yourself) และทำให้โค้ดอ่านง่ายขึ้น

  • ประกาศด้วย function functionName($param1, $param2 = 'default') { ... }
  • พารามิเตอร์ที่มีค่าเริ่มต้น (Default Value) ต้องอยู่ท้ายสุดเสมอ
  • ใช้ return เพื่อส่งค่ากลับ ฟังก์ชันจะหยุดทำงานเมื่อพบ return
  • PHP 7+ รองรับการกำหนดชนิดของ Parameter และ Return type: function add(int $a, int $b): int

Built-in Functions ที่ใช้บ่อย:

  • String: strlen(), strtolower(), strtoupper(), trim(), str_replace(), substr(), strpos()
  • Array: count(), array_push(), array_pop(), implode(), explode(), array_map(), array_filter(), sort(), in_array()
  • Math: abs(), round(), ceil(), floor(), rand(), pow(), sqrt()
  • Date: date(), time(), mktime(), strtotime()

การรับข้อมูลจากฟอร์ม HTML

HTML Form ใช้ส่งข้อมูลจากผู้ใช้ไปยัง PHP โดยมี 2 วิธีหลัก คือ GET และ POST กำหนดด้วย attribute method บน <form> และ action กำหนด URL ปลายทาง: 

┌────────────────────────────────────────────────────────────────┐
│              GET vs POST                                        │
├─────────────────────────┬──────────────────────────────────────┤
│          GET            │              POST                    │
├─────────────────────────┼──────────────────────────────────────┤
│ ข้อมูลใน URL            │ ข้อมูลใน Request Body               │
│ ?name=สมชาย&age=20     │ (ซ่อนจาก URL)                       │
├─────────────────────────┼──────────────────────────────────────┤
│ เห็นได้ใน Address Bar   │ ไม่เห็นใน Address Bar               │
│ สามารถ Bookmark ได้     │ Bookmark ไม่ได้                     │
├─────────────────────────┼──────────────────────────────────────┤
│ จำกัดความยาว ~2KB       │ ไม่จำกัดขนาด (เหมาะ Upload ไฟล์)  │
├─────────────────────────┼──────────────────────────────────────┤
│ เหมาะสำหรับ: ค้นหา,    │ เหมาะสำหรับ: Login, Register,       │
│ filter, pagination      │ บันทึกข้อมูล, Upload                 │
├─────────────────────────┼──────────────────────────────────────┤
│ PHP: $_GET['key']       │ PHP: $_POST['key']                  │
└─────────────────────────┴──────────────────────────────────────┘

Superglobals คือตัวแปรพิเศษของ PHP ที่เข้าถึงได้จากทุกที่ในโปรแกรม:

  • $_GET — ข้อมูลจาก Query String ใน URL
  • $_POST — ข้อมูลจาก Form ที่ใช้ method="POST"
  • $_REQUEST — รวมทั้ง GET และ POST (ไม่แนะนำ ใช้เฉพาะเจาะจงดีกว่า)
  • $_SESSION — ข้อมูลที่เก็บไว้ตลอด Session (ใช้หลัง session_start())
  • $_COOKIE — ข้อมูล Cookie จากเบราว์เซอร์
  • $_FILES — ไฟล์ที่ Upload ผ่านฟอร์ม
  • $_SERVER — ข้อมูล Server เช่น $_SERVER['REQUEST_METHOD']

การตรวจสอบและกรองข้อมูล (Validation & Sanitization)

ห้ามเชื่อถือข้อมูลจากผู้ใช้โดยตรง ผู้ใช้อาจส่งข้อมูลที่เป็นอันตรายมา เช่น HTML Tags, SQL Injection, หรือ Script ที่เป็นอันตราย เราต้องตรวจสอบและกรองข้อมูลทุกครั้ง:

  • isset() — ตรวจสอบว่าตัวแปรถูกกำหนดค่าและไม่เป็น null เช่น if (isset($_POST['name'])) { ... }
  • empty() — ตรวจสอบว่าตัวแปรว่างเปล่า ("", 0, null, [])
  • htmlspecialchars() — แปลง Special Characters เป็น HTML Entities ป้องกัน XSS: < → &lt;, > → &gt;
  • trim() — ลบ Whitespace หัวท้ายข้อความ ควรทำก่อน Validate ทุกครั้ง
  • filter_input() — กรองข้อมูลจาก Input: filter_input(INPUT_POST, 'email', FILTER_VALIDATE_EMAIL)
  • filter_var() — กรองและ Validate ค่าตัวแปร: filter_var($email, FILTER_VALIDATE_EMAIL)
ข้อควรระวัง — XSS Attack

หากไม่ใช้ htmlspecialchars() ผู้ใช้อาจส่ง <script>alert('Hacked!')</script> มาในฟอร์ม และ PHP จะแสดงโค้ดนั้นในหน้าเว็บ ทำให้ Script ทำงานและอาจขโมย Cookie หรือ Session ของผู้ใช้อื่นได้ ใช้ htmlspecialchars($input, ENT_QUOTES, 'UTF-8') เสมอ

🧪 ปฏิบัติการ Lab 3 — ตัวแปร, ฟังก์ชัน & รับข้อมูลฟอร์ม

ฝึกเขียนฟังก์ชัน PHP, ใช้ลูปสร้าง HTML, รับข้อมูลจาก GET/POST, และป้องกัน XSS

1
เขียนฟังก์ชันคำนวณ

สร้างไฟล์ functions_lab.php ที่มีฟังก์ชันคำนวณต่าง ๆ และทดสอบการเรียกใช้

  • เขียนฟังก์ชัน calculateBMI(float $weight, float $height): array ที่ส่งคืน array ที่มี bmi และ category
  • เขียนฟังก์ชัน gradeScore(int $score): string ที่ส่งคืนเกรด A/B/C/D/F
  • เขียนฟังก์ชัน calculateDiscount(float $price, float $percent): float คืนราคาหลังลด
  • ทดสอบทุกฟังก์ชันด้วยข้อมูลหลาย ๆ ชุด และแสดงผลในตาราง HTML
  • เพิ่ม Type Declarations ให้ทุกฟังก์ชัน (parameter types และ return type)
เกณฑ์การผ่าน

ฟังก์ชันทุกตัวทำงานถูกต้องกับ Input ที่หลากหลาย เช่น gradeScore(95) คืน 'A', calculateBMI(70, 1.75) คืน BMI ราว 22.86 หมวด 'ปกติ' ผลลัพธ์แสดงในตาราง HTML อย่างน้อย 5 แถวต่อฟังก์ชัน

คำแนะนำ

ใช้ match expression (PHP 8+) แทน switch สำหรับ gradeScore: return match(true) { $score >= 80 => 'A', $score >= 70 => 'B', ... }; ง่ายกว่าและไม่ต้องเขียน break

2
ใช้ลูปสร้างตารางสูตรคูณ

สร้างไฟล์ multiplication.php ที่ใช้ for loop สร้างตารางสูตรคูณ 1-12 เป็น HTML Table

  • แถวแรก (Header Row): แสดงตัวเลข 1-12 เป็น column headers
  • แถวต่อ ๆ ไป: แต่ละแถวคือสูตรคูณของตัวเลขนั้น (เช่น แถว 3 แสดง 3×1=3, 3×2=6, ...)
  • ใช้ nested loop: for ($row = 1; $row <= 12; $row++) และ for ($col = 1; $col <= 12; $col++)
  • Highlight เซลล์ที่ผลคูณเป็น Perfect Square (1, 4, 9, 16, ...) ด้วยสีพื้นหลังต่างกัน
  • เพิ่ม CSS ให้ตารางสวยงาม: Border, Alternating row colors, Font
เกณฑ์การผ่าน

ตารางแสดงผลถูกต้องทุกช่อง (12×12 = 144 เซลล์) มี Header ทั้งแถวและคอลัมน์ เซลล์ Perfect Square (เช่น 3×3=9, 4×4=16) มีสีพื้นหลังต่างออกไป

คำแนะนำ

ตรวจสอบ Perfect Square ด้วย: sqrt($row * $col) == floor(sqrt($row * $col)) หมายความว่าผลคูณของ row × col นั้นมีรากที่สองเป็นจำนวนเต็มพอดี

3
สร้างฟอร์มลงทะเบียน GET

สร้างไฟล์ register_get.php ที่มีฟอร์มลงทะเบียนโดยใช้ method GET

  • ฟอร์มมีช่อง: ชื่อ (name), อายุ (age), คณะ (faculty เป็น select)
  • ใช้ method="GET" ข้อมูลจะปรากฏใน URL หลัง Submit
  • รับค่าด้วย $_GET และแสดงผล: "สวัสดี [ชื่อ] จากคณะ [คณะ] อายุ [อายุ] ปี"
  • ตรวจสอบว่ามีการ Submit ก่อนแสดงผล (ใช้ isset($_GET['name']))
  • แสดง URL ที่สมบูรณ์ด้วย $_SERVER['REQUEST_URI'] เพื่อให้เห็นว่า GET ต่อ Query String
เกณฑ์การผ่าน

หลัง Submit ข้อมูลปรากฏใน URL เช่น ?name=สมชาย&age=20&faculty=engineering และหน้าเว็บแสดงผลที่ถูกต้อง หน้าก่อน Submit ต้องแสดงแค่ฟอร์ม ไม่มีข้อความแสดงผล

คำแนะนำ

เหมาะสำหรับ: ฟอร์มค้นหา, กรองข้อมูล, pagination เพราะผู้ใช้สามารถ Bookmark URL หรือ Copy & Share ได้ ไม่เหมาะสำหรับข้อมูลส่วนตัว (รหัสผ่าน, บัตรเครดิต) เพราะ URL บันทึกใน Browser History

4
สร้างฟอร์มติดต่อ POST

สร้างไฟล์ contact.php ที่มีฟอร์มติดต่อด้วย method POST พร้อม Validation ครบถ้วน

  • ฟอร์มมีช่อง: ชื่อ, Email, เบอร์โทร, หัวข้อ, ข้อความ ทุกช่องบังคับกรอก
  • Validate: ชื่อ (ไม่ว่าง), Email (format ถูกต้อง ด้วย filter_var), เบอร์โทร (10 หลัก), ข้อความ (อย่างน้อย 20 ตัวอักษร)
  • แสดง Error ใต้แต่ละ field ที่ผิดพลาด (สีแดง)
  • เมื่อ Submit สำเร็จ แสดงข้อมูลที่รับมาทั้งหมด และคงค่าเดิมไว้ในฟอร์มเมื่อมี Error
  • ใช้ htmlspecialchars() กับทุกค่าที่แสดงกลับในหน้าเว็บ
เกณฑ์การผ่าน

ทดสอบ: (1) Submit ว่าง — ต้องเห็น Error ทุก field, (2) กรอกถูกต้องทั้งหมด — แสดงผลสำเร็จ, (3) กรอก Email ผิด format — เห็น Error แค่ Email field อื่นผ่านปกติ

คำแนะนำ

Pattern สำหรับฟอร์ม PHP: ตรวจสอบด้วย if ($_SERVER['REQUEST_METHOD'] === 'POST') แทนการใช้ isset($_POST['submit']) เพราะผู้ใช้อาจลบ name attribute ออกจากปุ่ม Submit ได้

5
ป้องกัน XSS ด้วย htmlspecialchars

สร้างไฟล์ xss-demo.php เพื่อสาธิตความแตกต่างระหว่างการแสดงผลที่มีและไม่มีการป้องกัน XSS

  • สร้างฟอร์มง่าย ๆ: ช่องกรอกชื่อ 1 ช่อง และปุ่ม Submit
  • แสดงผลแบบ ไม่ปลอดภัย: echo "สวัสดี " . $_POST['name']; (ไม่ใช้ htmlspecialchars)
  • แสดงผลแบบ ปลอดภัย: echo "สวัสดี " . htmlspecialchars($_POST['name'], ENT_QUOTES, 'UTF-8');
  • ทดสอบโดยกรอก: <script>alert('XSS!')</script> ในช่องชื่อ
  • แสดงคำอธิบายผลที่ได้ทั้งสองกรณีในหน้าเว็บ
เกณฑ์การผ่าน

เมื่อกรอก <script>alert('XSS!')</script>: (1) แบบไม่ปลอดภัย — Script รันและเห็น Alert popup หรือเห็นว่า HTML ถูก Render เป็น Script tag ใน Source, (2) แบบปลอดภัย — แสดงข้อความ literal <script>alert('XSS!')</script> โดยไม่รัน Script

คำแนะนำ

ดู HTML Source ด้วย Ctrl+U (View Page Source) เพื่อเห็นความแตกต่าง แบบไม่ปลอดภัยจะเห็น <script> จริง ๆ ใน Source แบบปลอดภัยจะเห็น &lt;script&gt; ซึ่งเบราว์เซอร์จะแสดงเป็นข้อความ ไม่รันเป็น Script
