การทดสอบเว็บแอปพลิเคชัน (Testing)

ก่อน Deploy ระบบขึ้นสู่สาธารณะ สิ่งสำคัญที่สุดคือการทดสอบอย่างรอบคอบ บัคที่หาไม่เจอตอนพัฒนาอาจกลายเป็นปัญหาใหญ่เมื่อผู้ใช้จริงเจอมัน ประเภทของการทดสอบที่ควรทำ:

  • Functional Testing (ทดสอบฟังก์ชันการทำงาน) — ระบบทำงานได้ตาม Requirement หรือไม่? ทดสอบทุก Feature: Login, CRUD, Upload, Search, ฯลฯ
  • Usability Testing (ทดสอบความสะดวกในการใช้) — ให้คนอื่นที่ไม่ได้ทำโปรเจกต์ลองใช้ แล้วสังเกตว่าเขาสับสนตรงไหน ปุ่มหรือเมนูไม่ชัดเจนตรงไหน
  • Security Testing (ทดสอบความปลอดภัย) — ลอง SQL Injection ในช่องค้นหา, ลอง XSS ในช่อง Comment, ลองเข้าหน้า Admin โดยไม่ Login
  • Browser Compatibility Testing — ทดสอบบน Chrome, Firefox, Edge, Safari และบน Mobile (iOS Safari, Android Chrome)
  • Performance Testing — ระบบช้าหรือเปล่าเมื่อมีข้อมูลมาก? ลองใส่สินค้า 1,000 รายการ แล้วดูว่าหน้าโหลดเร็วพอไหม

Checklist สิ่งที่ต้องทดสอบก่อน Deploy: 

การทดสอบก่อน Deploy — Checklist

Authentication:
  [ ] Login ด้วย username/password ถูก → เข้าได้
  [ ] Login ด้วย password ผิด → แสดง error ที่เหมาะสม
  [ ] เข้าหน้า Admin โดยไม่ Login → Redirect ไป Login
  [ ] Session Timeout ทำงานหรือไม่?

Form & Input:
  [ ] Submit form ว่างเปล่า → แสดง validation error
  [ ] ใส่ตัวอักษรพิเศษ/HTML ในช่อง input → ไม่ทำให้หน้าพัง
  [ ] อัปโหลดไฟล์ขนาดใหญ่เกินกำหนด → แสดง error ที่ชัดเจน
  [ ] อัปโหลดไฟล์นามสกุลไม่ถูกต้อง → ปฏิเสธและแจ้งเตือน

Database:
  [ ] บันทึกข้อมูลลง DB → ข้อมูลถูกต้องใน phpMyAdmin
  [ ] แก้ไขข้อมูล → อัปเดตใน DB จริง
  [ ] ลบข้อมูล → หายจาก DB และไม่แสดงในหน้าเว็บ
  [ ] ค้นหาด้วยคำที่ไม่มีในระบบ → แสดง "ไม่พบข้อมูล"

Mobile:
  [ ] แสดงผลถูกต้องบน Mobile (ไม่ overflow, ไม่ตัดข้อความ)
  [ ] ปุ่มกดได้ง่ายบน touch screen
  [ ] Form กรอกได้สะดวกบน keyboard ของ mobile
เคล็ดลับ

ให้เพื่อนหรือคนในครอบครัวที่ไม่รู้เรื่อง Web ลองใช้ระบบของคุณ แล้วสังเกตว่าเขาสับสนตรงไหน นักพัฒนามักมองข้าม UX ที่แย่เพราะคุ้นชินกับระบบของตัวเอง ผู้ใช้จริงจะพบปัญหาที่คุณไม่เคยนึกถึง

การ Deploy ขึ้น Shared Hosting

Shared Hosting คือบริการ Hosting ที่ผู้ให้บริการมีเซิร์ฟเวอร์ขนาดใหญ่ และแบ่งพื้นที่ให้ลูกค้าหลายรายใช้ร่วมกัน เป็นตัวเลือกราคาถูกที่เหมาะสำหรับเว็บขนาดเล็กถึงกลาง ผู้ให้บริการยอดนิยม: Hostinger, SiteGround, Namecheap, หรือ HostArmada

ขั้นตอนการ Deploy บน Shared Hosting:

  1. อัปโหลดไฟล์ด้วย FTP — ดาวน์โหลด FileZilla (ฟรี) เชื่อมต่อด้วย FTP credentials จาก Hosting Panel แล้ว Upload ไฟล์ทั้งหมดไปยัง public_html/
  2. สร้าง Database บน Hosting — ใน cPanel หรือ Hosting Panel ไปที่ MySQL Databases สร้าง Database ใหม่, User ใหม่, และกำหนด privileges
  3. Import Database — ใน phpMyAdmin บน Hosting เลือก Import แล้วเลือกไฟล์ database.sql ที่เตรียมไว้
  4. แก้ไข config.php — เปลี่ยน DB_HOST, DB_USER, DB_PASS, DB_NAME เป็นค่าของ Hosting (ไม่ใช่ localhost อีกต่อไป) โดย Hosting มักให้ HOST เป็น localhost หรือ IP ภายใน
  5. ตรวจสอบ PHP Version — Hosting บางแห่งใช้ PHP เวอร์ชันเก่า ตรวจสอบใน cPanel ว่า PHP version ตรงกับที่พัฒนา (แนะนำ PHP 8.0+)
  6. ตั้งค่า File Permissions — โฟลเดอร์ uploads/ ต้องให้ Web Server เขียนได้ ตั้งเป็น 755 หรือ 775 ผ่าน FTP client หรือ File Manager ใน cPanel
ข้อควรระวัง

ห้ามอัปโหลด config.php ที่มีรหัสผ่านจริงขึ้น Public Repository (GitHub) ให้ใช้ FTP อัปโหลดโดยตรง หรือ เก็บ credentials ใน .env file และเพิ่ม config.php ใน .gitignore

การ Deploy บน Free Platforms

สำหรับนักศึกษาที่ต้องการ Deploy โดยไม่เสียค่าใช้จ่าย มีตัวเลือกดังนี้:

  • InfinityFree (infinityfree.net) — Hosting PHP + MySQL ฟรี มีข้อจำกัดเรื่อง CPU และ Bandwidth แต่เพียงพอสำหรับโปรเจกต์นักศึกษา รองรับ PHP 7.4-8.1
  • 000WebHost (Hostinger) — ฟรี Hosting PHP + MySQL 1GB มีข้อจำกัดเรื่องเวลาทำงาน (Sleep หลังไม่ใช้งานนาน)
  • Railway.app — รองรับ PHP, MySQL, PostgreSQL ฟรีในระดับ Hobby เหมาะสำหรับนักศึกษาที่ต้องการ Deploy แบบมืออาชีพมากขึ้น
  • Render.com — Web Service ฟรี (sleep หลัง 15 นาทีไม่ใช้งาน) รองรับ PHP ผ่าน Docker หรือ Dockerfile ต้องการความรู้เพิ่มเติม
  • GitHub Pages + PHP Hosting — ใช้ GitHub Pages สำหรับ Static Frontend และ Free PHP Hosting สำหรับ API Backend (Split Architecture)
หมายเหตุ

Free Hosting มักมีข้อจำกัดและอาจไม่ Stable สำหรับโปรเจกต์ Production จริง แต่เพียงพอสำหรับการ Demo และ Portfolio เมื่อต้องการ Deploy จริงๆ ในอนาคต ควรลงทุนกับ Paid Hosting ที่เริ่มต้นที่ประมาณ 50-100 บาท/เดือน

ความปลอดภัยก่อน Deploy

ก่อนให้ผู้ใช้จริงเข้าถึงระบบ ต้องตรวจสอบความปลอดภัยขั้นพื้นฐาน:

  • ลบข้อมูลทดสอบ — ลบ user ทดสอบ, ข้อมูลตัวอย่างที่ไม่จริง, และ debug output ทั้งหมด
  • เปลี่ยนรหัสผ่าน Default — MySQL root password, admin account ใน web ห้ามใช้ "password", "1234", "admin" ในระบบ Production
  • ปิดการแสดง Error — ตั้ง display_errors = Off ใน php.ini หรือในโค้ด PHP Error message ที่แสดงต่อผู้ใช้อาจเปิดเผยโครงสร้าง DB หรือ Path ระบบ
  • ใช้ Environment Variables — แทนที่จะ hardcode credentials ใน config.php ใช้ $_ENV หรือ getenv() ดึงจาก .env file หรือ Hosting Environment
  • HTTPS เสมอ — Hosting ทุกแห่งปัจจุบันให้ SSL Certificate ฟรี (Let's Encrypt) บังคับ HTTPS ด้วย .htaccess redirect
  • Prepared Statements ทุกที่ — ตรวจสอบว่าทุก SQL Query ใช้ Prepared Statements ไม่มี String concatenation ใน SQL เลย
  • Validate ทุก Input — ทั้ง Client-side (JavaScript) และ Server-side (PHP) อย่าเชื่อข้อมูลที่ Client ส่งมาโดยไม่ตรวจสอบ 
Security Checklist ก่อน Deploy

  [ ] display_errors = Off ใน Production
  [ ] ไม่มี var_dump() หรือ print_r() ในโค้ด
  [ ] ทุก SQL ใช้ Prepared Statements (PDO/MySQLi)
  [ ] ทุก Output ใช้ htmlspecialchars() (ป้องกัน XSS)
  [ ] รหัสผ่านเก็บด้วย password_hash() (bcrypt)
  [ ] Session ใช้ session_regenerate_id() หลัง Login
  [ ] ไฟล์ config.php ไม่อยู่ใน public_html โดยตรง
  [ ] โฟลเดอร์ uploads/ ไม่อนุญาต Execute PHP
  [ ] .htaccess ปิดการเข้าถึงไฟล์ .sql .env .log
  [ ] HTTPS เปิดใช้งาน (SSL Certificate)

ทักษะเหล่านี้คือรากฐานของนักพัฒนาเว็บทุกคน ไม่ว่าจะทำงานใน Startup เล็กๆ หรือบริษัทใหญ่ระดับ Enterprise ทุกเว็บแอปพลิเคชันบน Internet ใช้หลักการเดียวกัน: HTML/CSS/JS สำหรับ Frontend, Backend Language (PHP, Python, Node.js ฯลฯ) สำหรับ Logic, และ Database สำหรับเก็บข้อมูล

เชื่อมโยงกับอุตสาหกรรม: ทักษะที่เรียนมาสัมพันธ์โดยตรงกับตำแหน่งงาน:

  • Web Developer (Junior) — สร้างหน้าเว็บ, แก้บัค, ทำงานตาม Spec ที่กำหนด
  • Full Stack Developer — ทำได้ทั้ง Frontend และ Backend เหมือนที่เรียนมาทั้งหมด
  • PHP Developer — พัฒนา Backend ด้วย PHP ใช้ Framework เช่น Laravel
  • Database Administrator — ออกแบบ ดูแล และ Optimize ฐานข้อมูล

deploy-checklist.sh Bash 
#!/bin/bash
# deploy-checklist.sh — สคริปต์ตรวจสอบก่อน Deploy

echo "=== Pre-Deploy Checklist ==="

# ── ตรวจสอบ PHP Syntax ทุกไฟล์ ───────────────────────────────
echo ""
echo "1. ตรวจสอบ PHP Syntax..."
find . -name "*.php" -not -path "./vendor/*" | while read file; do
    php -l "$file" 2>&1 | grep -v "No syntax errors"
done
echo "   เสร็จ!"

# ── ตรวจสอบ var_dump / print_r ที่อาจลืมลบ ──────────────────
echo ""
echo "2. ตรวจสอบ Debug Output ที่ลืมลบ..."
DEBUG=$(grep -rn "var_dump\|print_r\|die(\|exit(" --include="*.php" . \
        --exclude-dir=vendor)
if [ -n "$DEBUG" ]; then
    echo "   พบ Debug output:"
    echo "$DEBUG"
else
    echo "   ไม่พบ Debug output ✓"
fi

# ── ตรวจสอบ display_errors ──────────────────────────────────
echo ""
echo "3. ตรวจสอบ display_errors..."
DE=$(grep -rn "display_errors\s*=\s*On\|display_errors\s*=\s*1" \
     --include="*.php" .)
if [ -n "$DE" ]; then
    echo "   พบ display_errors = On — ต้องปิดก่อน Deploy:"
    echo "$DE"
else
    echo "   display_errors ถูกปิด หรือไม่ได้ตั้งค่าชัดเจน ✓"
fi

# ── ตรวจสอบ File Permissions ─────────────────────────────────
echo ""
echo "4. ตรวจสอบ Permissions..."
if [ -d "uploads" ]; then
    UPLOADS_PERM=$(stat -c "%a" uploads/ 2>/dev/null || stat -f "%p" uploads/)
    echo "   uploads/ permission: $UPLOADS_PERM"
    echo "   แนะนำ: 755 (Apache สามารถเขียนได้, ผู้ใช้อื่นอ่านได้)"
fi

# ── ตรวจสอบ Hardcoded Credentials ───────────────────────────
echo ""
echo "5. ตรวจสอบ Hardcoded Credentials..."
CREDS=$(grep -rn "password.*=.*['\"][^'\"]*['\"]" --include="*.php" . \
        | grep -v "password_hash\|password_verify\|config.php")
if [ -n "$CREDS" ]; then
    echo "   อาจพบ Hardcoded password (ตรวจสอบด้วยตนเอง):"
    echo "$CREDS" | head -5
else
    echo "   ไม่พบ Hardcoded password ชัดเจน ✓"
fi

echo ""
echo "=== Checklist เสร็จสิ้น ==="
echo "ตรวจสอบผลลัพธ์ด้านบนและแก้ไขปัญหาก่อน Deploy"

    // Production: แสดง error ที่เป็นมิตร ไม่เปิดเผยรายละเอียด

  • Laravel — PHP Framework ที่ได้รับความนิยมสูงสุด ทำทุกอย่างที่เรียนมาได้เร็วกว่าและปลอดภัยกว่า: Routing, ORM (Eloquent), Authentication สำเร็จรูป, Queue, Mail, ฯลฯ
  • React.js — JavaScript Library สำหรับ Front-end สมัยใหม่ สร้าง UI แบบ Component-based ทำงานร่วมกับ REST API ที่เรียนมาได้ทันที
  • Git & GitHub — ระบบควบคุมเวอร์ชัน จำเป็นสำหรับนักพัฒนาทุกคน ทำงานเป็นทีมได้อย่างมีประสิทธิภาพ, Portfolio บน GitHub คือ Resume ที่ดีที่สุด
  • Docker — Containerization สำหรับ Deploy แบบมืออาชีพ แก้ปัญหา "ทำงานบนเครื่องฉันได้แต่บน Server ไม่ได้" ได้ถาวร
  • Vue.js หรือ Next.js — ทางเลือกอื่นของ JavaScript Framework ที่ได้รับความนิยมสูง
  • Linux / Command Line — นักพัฒนาส่วนใหญ่ทำงานบน Linux Server ความชำนาญ Terminal ช่วยได้มาก
แหล่งเรียนรู้ที่แนะนำ
  • php.net — Official PHP Documentation ครบถ้วนและมี Examples ดี
  • laracasts.com — วิดีโอสอน Laravel และ PHP Modern ที่ดีที่สุด
  • w3schools.com — Reference สำหรับ HTML, CSS, JS, PHP, SQL
  • stackoverflow.com — ถาม-ตอบปัญหา Programming ที่ใหญ่ที่สุดในโลก
  • freecodecamp.org — คอร์สเรียน Web Development ฟรีที่มีคุณภาพสูง
  • youtube.com/Traversy Media — วิดีโอสอน Web Development ภาษาอังกฤษที่ดีมาก
