ทำไมต้องเขียนโค้ดที่ใช้ซ้ำได้?

ในการพัฒนาเว็บแอปพลิเคชันจริง โปรเจกต์มักประกอบด้วยหน้าเว็บหลายสิบหรือหลายร้อยหน้า ถ้าเราคัดลอก (copy-paste) โค้ดเดิมซ้ำๆ ในแต่ละหน้า เช่น ส่วน Header, Navbar, และ Footer จะเกิดปัญหาใหญ่ตามมา: เมื่อต้องการแก้ไขโลโก้หรือเมนูนำทาง เราต้องแก้ไขทุกหน้าพร้อมกัน ซึ่งหมายถึงการแก้บัคในที่เดียวกันอาจต้องทำซ้ำถึง 10 หรือ 20 ที่ ทำให้เกิดข้อผิดพลาดได้ง่าย

หลักการสำคัญในการพัฒนาซอฟต์แวร์ที่แก้ปัญหานี้คือ DRY — Don't Repeat Yourself ซึ่งหมายความว่าโค้ดแต่ละชิ้นควรมีอยู่เพียงที่เดียว และถูกนำมาใช้ซ้ำในที่ที่ต้องการ แทนที่จะก็อปปี้ การทำตามหลัก DRY ช่วยให้โค้ดของเราดูแลรักษาง่ายขึ้น (Maintainability) มีความสม่ำเสมอ (Consistency) และลดโอกาสเกิดบัคที่เกิดจากการแก้ไขไม่ครบทุกที่

  • Maintainability — แก้ไขโค้ดที่เดียว มีผลทั่วทั้งระบบ
  • Consistency — ทุกหน้าใช้ Header และ Footer แบบเดียวกัน ไม่มีความแตกต่าง
  • Readability — ไฟล์แต่ละหน้ามีขนาดเล็กลง อ่านง่ายขึ้น โฟกัสที่เนื้อหาของหน้านั้นๆ
  • Reusability — สร้าง Component ครั้งเดียว ใช้ได้ทั่วทั้งโปรเจกต์
หมายเหตุ

ปัญหาของการ Copy-Paste โค้ด: ลองนึกภาพว่าคุณมีเว็บไซต์ 20 หน้า แต่ละหน้าก็อปปี้ Navbar มาเอง วันหนึ่งต้องเพิ่มเมนูใหม่ คุณต้องแก้ไข 20 ไฟล์พร้อมกัน และถ้าลืมแก้แค่ 1 ไฟล์ ผู้ใช้ที่เข้าหน้านั้นจะเห็น Navbar ที่ไม่สมบูรณ์ — นี่คือปัญหาที่ DRY แก้ได้

PHP include และ require

PHP มีคำสั่งในตระกูล include และ require ที่ใช้แทรกเนื้อหาจากไฟล์อื่นเข้ามาในไฟล์ปัจจุบัน ทำให้เราแบ่งโค้ดออกเป็น Component ย่อยๆ ได้อย่างสะดวก

  • include — แทรกไฟล์เข้ามา ถ้าไม่พบไฟล์จะแสดง Warning แต่สคริปต์ยังทำงานต่อได้
  • require — แทรกไฟล์เข้ามา ถ้าไม่พบไฟล์จะเกิด Fatal Error และหยุดการทำงานทันที เหมาะสำหรับไฟล์ที่จำเป็นต้องมี เช่น config.php
  • include_once / require_once — เหมือนกันกับข้างต้น แต่ตรวจสอบก่อนว่าเคยแทรกไฟล์นี้แล้วหรือยัง ถ้าแล้วจะข้ามไป ป้องกันการประกาศฟังก์ชันซ้ำซ้อน

โครงสร้างไฟล์ที่นิยมใช้ในโปรเจกต์ PHP: 

project/
├── config/
│   └── config.php          ← ค่าการตั้งค่าและค่าคงที่ (DB, Site URL, ฯลฯ)
├── includes/
│   ├── header.php          ← DOCTYPE, <head>, <nav> ส่วนบนของทุกหน้า
│   ├── footer.php          ← <footer>, <script> ส่วนล่างของทุกหน้า
│   ├── navbar.php          ← เมนูนำทาง (อาจรวมอยู่ใน header.php)
│   └── functions.php       ← ฟังก์ชัน Helper ที่ใช้ทั่วทั้งโปรเจกต์
├── index.php               ← หน้าหลัก (require header, content, footer)
├── products.php            ← หน้าสินค้า (require header, content, footer)
└── about.php               ← หน้าเกี่ยวกับ (require header, content, footer)
เคล็ดลับ

ใช้ require_once สำหรับ config.php และ functions.php เสมอ เพราะถ้าไม่มีไฟล์เหล่านี้เว็บจะทำงานไม่ได้เลย และ _once ป้องกันการประกาศฟังก์ชันซ้ำ ในกรณีที่มีการ include ซ้อนกัน (nested include)

Template Pattern ด้วย PHP

Template Pattern คือรูปแบบการออกแบบที่แยกส่วน Layout (โครงร่างหน้าเว็บ) ออกจากเนื้อหา (Content) แทนที่แต่ละหน้าจะ include header/footer เอง เราสร้างไฟล์ layout.php ที่เป็นโครงร่างหลัก แล้วฉีด (inject) เนื้อหาเข้าไปแทน มี 2 วิธีหลัก:

  • ผ่านตัวแปร — ตั้งค่า $title และ $content ก่อนแล้วค่อย include layout.php
  • ผ่าน Output Buffering — ใช้ ob_start() และ ob_get_clean() จับ HTML ที่ถูกสร้างขึ้นใส่ตัวแปรก่อนส่งไปยัง layout

การแยก Logic ออกจาก Presentation เป็นแนวคิดสำคัญในสถาปัตยกรรมซอฟต์แวร์ ช่วยให้ Front-end developer แก้ไขหน้าตาได้โดยไม่ต้องยุ่งกับ Business Logic และ Back-end developer แก้ Logic ได้โดยไม่ต้องกังวลเรื่อง HTML

ข้อควรระวัง

เมื่อใช้ Output Buffering ต้องแน่ใจว่าเรียก ob_end_clean() หรือ ob_get_clean() ทุกครั้งที่เรียก ob_start() เพราะถ้าลืมปิด Buffer อาจทำให้หน้าเว็บแสดงผลผิดพลาดหรือข้อมูลรั่วไหลระหว่างหน้า

ฟังก์ชัน Helper

ฟังก์ชัน Helper คือฟังก์ชันที่เราสร้างขึ้นมาเองเพื่อทำงานที่ใช้บ่อยในโปรเจกต์ รวบรวมไว้ใน includes/functions.php แล้ว include ในทุกหน้า ตัวอย่างฟังก์ชัน Helper ที่มีประโยชน์:

  • formatThaiDate($date) — แปลงวันที่จาก Y-m-d เป็นรูปแบบ "28 มิถุนายน 2568"
  • formatPrice($amount) — จัดรูปแบบตัวเลขเป็นราคา เช่น "1,250.00 บาท"
  • slugify($text) — แปลงข้อความเป็น URL-friendly เช่น "สินค้าใหม่" → "sinkha-mai"
  • truncateText($text, $limit) — ตัดข้อความให้สั้นลง เพิ่ม "..." ที่ท้าย
  • sanitize($input) — ทำความสะอาด Input เพื่อป้องกัน XSS
  • redirect($url) — Redirect ไปยัง URL อื่นพร้อม header()

การรวม Helper ทั้งหมดไว้ในไฟล์เดียวทำให้เราแน่ใจว่าทุกหน้าใช้ฟังก์ชันเดียวกัน และถ้าต้องการอัปเดตการ format วันที่หรือราคา ก็แก้แค่ที่เดียว

Class และ Object พื้นฐาน (OOP)

Object-Oriented Programming (OOP) เป็นรูปแบบการเขียนโปรแกรมที่จัดระเบียบโค้ดในรูปแบบ "วัตถุ" (Object) แต่ละวัตถุมี Properties (คุณลักษณะ) และ Methods (พฤติกรรม) PHP รองรับ OOP ตั้งแต่เวอร์ชัน 5 และปัจจุบัน Framework ขนาดใหญ่อย่าง Laravel ใช้ OOP เป็นหลัก

  • Class — แบบพิมพ์เขียว (Blueprint) ที่กำหนดโครงสร้างของวัตถุ
  • Object — instance ที่สร้างจาก Class โดยใช้ new
  • Property — ตัวแปรที่อยู่ใน Class เก็บข้อมูลของวัตถุ
  • Method — ฟังก์ชันที่อยู่ใน Class กำหนดพฤติกรรมของวัตถุ
  • Constructor — Method พิเศษ __construct() ที่ถูกเรียกอัตโนมัติเมื่อสร้าง Object
  • Access Modifiers — public (เข้าถึงได้จากทุกที่), private (เฉพาะใน Class), protected (ใน Class และ Subclass)

หมายเหตุ

OOP ไม่ใช่สิ่งที่ต้องใช้ในทุกส่วนของโปรเจกต์ขนาดเล็ก แต่การเรียนรู้พื้นฐาน Class และ Object จะเตรียมความพร้อมสำหรับการใช้ Framework เช่น Laravel ซึ่งใช้ OOP ทุกที่ และช่วยให้เข้าใจ Library และ Package ต่างๆ ที่เราจะใช้ในอนาคต

/**
 * ตัดข้อความให้สั้นลงและเพิ่ม "..." ที่ท้าย
 * @param string $text  — ข้อความต้นฉบับ
 * @param int    $limit — จำนวนตัวอักษรสูงสุด
 * @return string
 */
function limitText(string $text, int $limit = 100): string {
    if (mb_strlen($text) <= $limit) {
        return $text;
    }
    return mb_substr($text, 0, $limit) . '...';
}

}

    if ($hour >= 12 && $hour < 17) return 'สวัสดีตอนบ่าย';

    // Method: คืนราคาในรูปแบบที่อ่านง่าย
    public function getFormattedPrice(): string {
        return '฿' . number_format($this->price, 2);
    }

    // Method: ตรวจสอบว่ายังมีสินค้าในสต็อกหรือไม่
    public function isInStock(): bool {
        return $this->stock > 0;
    }

    // Method: ลดจำนวนสต็อก
    public function decreaseStock(int $quantity = 1): bool {
        if ($this->stock < $quantity) {
            return false; // สต็อกไม่พอ
        }
        $this->stock -= $quantity;
        return true;
    }

    // Method: คืนข้อมูลเป็น Array
    public function toArray(): array {
        return [
            'id'    => $this->id,
            'name'  => $this->name,
            'price' => $this->price,
            'stock' => $this->stock,
            'in_stock' => $this->isInStock(),
        ];

🧪 ปฏิบัติการ Lab 10 — Code Reusability & Template

นำหลักการ DRY มาใช้กับโปรเจกต์ของคุณ โดยแยกส่วนที่ใช้ซ้ำออกเป็น Component และสร้าง Template System

1
แยกไฟล์ Header และ Footer

นำโปรเจกต์จาก Lab สัปดาห์ก่อน (หรือสร้างโปรเจกต์ใหม่ที่มีอย่างน้อย 3 หน้า) แล้วทำดังนี้:

  1. สร้างโฟลเดอร์ includes/ ในโปรเจกต์
  2. สร้างไฟล์ includes/header.php — ย้ายส่วน HTML ตั้งแต่ <!DOCTYPE html> จนถึง <main> เข้ามาในไฟล์นี้
  3. สร้างไฟล์ includes/footer.php — ย้ายส่วน </main>, <footer>, และ </html> เข้ามา
  4. แทนที่โค้ดที่ย้ายออกไปด้วย <?php require_once 'includes/header.php'; ?> และ <?php require_once 'includes/footer.php'; ?> ในทุกหน้า
  5. ทดสอบว่าทุกหน้ายังแสดงผลถูกต้อง
เกณฑ์การผ่าน

ทุกหน้าของโปรเจกต์แสดงผลถูกต้อง Header และ Footer แสดงเหมือนกันทุกหน้า โดยไม่มีโค้ด Header/Footer ซ้ำในไฟล์หลัก

คำแนะนำ

ใช้ require_once แทน include สำหรับ Header และ Footer เพราะถ้าไม่มีไฟล์นี้หน้าเว็บจะพังทั้งหมด การใช้ require ทำให้เกิด Fatal Error ที่ชัดเจนกว่า Warning

2
สร้าง config.php

สร้างไฟล์ config/config.php ที่รวบรวมค่าการตั้งค่าของโปรเจกต์ทั้งหมดไว้ที่เดียว:

  1. สร้างโฟลเดอร์ config/ และไฟล์ config/config.php
  2. กำหนด Constants สำหรับ Database: DB_HOST, DB_USER, DB_PASS, DB_NAME
  3. กำหนด Constants สำหรับ Site: SITE_NAME, SITE_URL, UPLOAD_DIR
  4. ย้ายการเชื่อมต่อฐานข้อมูลมาไว้ใน config.php และสร้างตัวแปร $pdo ให้พร้อมใช้
  5. แทนที่การเชื่อมต่อฐานข้อมูลในทุกหน้าด้วย require_once 'config/config.php';
เกณฑ์การผ่าน

ทุกหน้าในโปรเจกต์ใช้ $pdo จาก config.php ได้อย่างถูกต้อง ไม่มีการ hardcode รหัสผ่านหรือชื่อฐานข้อมูลซ้ำในไฟล์อื่น

คำแนะนำ

กำหนด Constants ด้วย define('DB_HOST', 'localhost'); แทนตัวแปร $DB_HOST เพราะ Constants สามารถเข้าถึงได้ทุกที่โดยไม่ต้อง global และไม่สามารถถูก override ได้โดยบังเอิญ

3
สร้าง functions.php

สร้างไฟล์ includes/functions.php พร้อมฟังก์ชัน Helper ที่ใช้บ่อย:

  1. เขียนฟังก์ชัน formatThaiCurrency(float $amount): string — คืนราคาในรูปแบบ "฿1,250.00"
  2. เขียนฟังก์ชัน getGreeting(): string — คืนคำทักทายตามช่วงเวลา (เช้า/บ่าย/เย็น/ดึก)
  3. เขียนฟังก์ชัน limitText(string $text, int $limit = 100): string — ตัดข้อความและเพิ่ม "..."
  4. Include functions.php ใน header.php เพื่อให้ทุกหน้าสามารถใช้ฟังก์ชันเหล่านี้ได้
  5. ทดสอบการใช้ฟังก์ชันในหน้าสินค้า: แสดงราคาด้วย formatThaiCurrency()
เกณฑ์การผ่าน

ฟังก์ชันทั้งสามทำงานถูกต้อง: formatThaiCurrency(1250) คืน "฿1,250.00", getGreeting() คืนคำทักทายที่เหมาะสมกับเวลาปัจจุบัน, limitText("ข้อความยาว", 5) คืน "ข้อความ..."

คำแนะนำ

ใช้ mb_strlen() และ mb_substr() แทน strlen() และ substr() สำหรับข้อความภาษาไทย เพราะฟังก์ชัน mb_ รองรับ Multi-byte characters (UTF-8) อย่างถูกต้อง

4
ใช้ Template Pattern

สร้าง Master Layout ที่ใช้ร่วมกันทั้งโปรเจกต์ด้วย Template Pattern:

  1. สร้างไฟล์ includes/layout.php ที่มีโครงสร้าง HTML สมบูรณ์ รับตัวแปร $title และ $content
  2. แปลงหน้า index.php ให้ใช้ Output Buffering: ใช้ ob_start() จับเนื้อหา แล้ว require_once 'includes/layout.php'
  3. ทำซ้ำกับอีก 2 หน้า (เช่น products.php, about.php)
  4. ทดสอบว่าแต่ละหน้าแสดง <title> ที่แตกต่างกันตาม $title ที่กำหนด
เกณฑ์การผ่าน

ทั้ง 3 หน้าใช้ layout.php ร่วมกัน แต่ละหน้ามี <title> ที่แตกต่างกัน และเนื้อหาของแต่ละหน้าแสดงถูกต้องในส่วน $content

คำแนะนำ

ลำดับที่ถูกต้อง: (1) กำหนด $title, (2) เรียก ob_start(), (3) เขียน HTML เนื้อหา, (4) $content = ob_get_clean(), (5) require_once layout.php ถ้าลำดับผิดจะทำให้ Layout แสดงก่อนเนื้อหาหรือแสดงซ้อนกัน

5
สร้าง Class ง่ายๆ

สร้าง PHP Class สำหรับจัดการข้อมูลสินค้าในโปรเจกต์:

  1. สร้างโฟลเดอร์ classes/ และไฟล์ classes/Product.php
  2. สร้าง Class Product มี Properties: $id, $name, $price, $stock
  3. เพิ่ม Constructor ที่รับ id, name, price, และ stock (default = 0)
  4. เพิ่ม Method getFormattedPrice() คืนราคาในรูปแบบ "฿X,XXX.XX"
  5. เพิ่ม Method isInStock() คืน true/false ตามจำนวนสต็อก
  6. ในหน้า products.php: include Product.php, ดึงข้อมูลจาก DB, สร้าง Object Product สำหรับแต่ละแถว, แสดงผลด้วย Method ของ Class
เกณฑ์การผ่าน

หน้า products.php แสดงรายการสินค้าโดยใช้ Object ของ Class Product แต่ละสินค้าแสดงราคาผ่าน getFormattedPrice() และมี badge "หมดแล้ว" สำหรับสินค้าที่ isInStock() คืน false

คำแนะนำ

เมื่อดึงข้อมูลจาก Database ด้วย PDO ให้ใช้ new Product($row['id'], $row['name'], $row['price'], $row['stock']) สร้าง Object ในแต่ละ iteration ของ loop แล้วเก็บใน array เพื่อแสดงผล
