พื้นฐานฐานข้อมูล Relational Database

ฐานข้อมูลเชิงสัมพันธ์ (Relational Database) เก็บข้อมูลในรูปแบบ ตาราง (Table) ซึ่งประกอบด้วย แถว (Row/Record) และ คอลัมน์ (Column/Field) แต่ละตารางจะมีชื่อเฉพาะ และเชื่อมโยงกับตารางอื่นผ่าน Key

แนวคิดสำคัญของ Relational Database:

  • Primary Key (PK) — คอลัมน์ที่ระบุ Record อย่างเป็นเอกลักษณ์ ห้ามซ้ำและห้ามเป็น NULL นิยมใช้คอลัมน์ id แบบ INT AUTO_INCREMENT
  • Foreign Key (FK) — คอลัมน์ที่อ้างอิง Primary Key ของตารางอื่น ใช้สร้างความสัมพันธ์ระหว่างตาราง เช่น orders.user_id อ้างอิง users.id
  • Index — โครงสร้างข้อมูลพิเศษที่ช่วยให้ค้นหาข้อมูลเร็วขึ้น (คล้ายดัชนีหนังสือ) Primary Key จะมี Index อัตโนมัติ
  • Constraint — กฎที่บังคับกับข้อมูล เช่น NOT NULL, UNIQUE, DEFAULT, CHECK

ชนิดข้อมูล MySQL ที่ใช้บ่อย:

  • INT — จำนวนเต็ม (-2 ล้าน ถึง +2 ล้าน), BIGINT — จำนวนเต็มขนาดใหญ่
  • VARCHAR(n) — ข้อความความยาวแปรผัน ไม่เกิน n ตัวอักษร (เหมาะสำหรับ ชื่อ, Email)
  • TEXT — ข้อความยาว ไม่จำกัด (เหมาะสำหรับ รายละเอียด, เนื้อหา)
  • DECIMAL(p,s) — ทศนิยมแม่นยำสูง (เช่น DECIMAL(10,2) สำหรับราคา)
  • FLOAT / DOUBLE — ทศนิยม (ความแม่นยำน้อยกว่า DECIMAL)
  • BOOLEAN / TINYINT(1) — จริง/เท็จ (0 หรือ 1)
  • DATE — วันที่ (YYYY-MM-DD), TIME — เวลา, DATETIME — วันที่และเวลา
  • TIMESTAMP — เหมือน DATETIME แต่อ้างอิง UTC อัปเดตอัตโนมัติได้
  • ENUM('val1','val2',...) — ค่าที่กำหนดไว้ล่วงหน้า (เช่น สถานะ)

การออกแบบ ER Diagram

ER Diagram (Entity-Relationship Diagram) คือแผนภาพที่แสดง Entity (ตาราง), Attribute (คอลัมน์), และ Relationship (ความสัมพันธ์) ระหว่าง Entity ใช้ออกแบบโครงสร้างฐานข้อมูลก่อนสร้างจริง

ประเภทของ Relationship:

  • 1:1 (One-to-One) — หนึ่ง Record ใน Table A สัมพันธ์กับหนึ่ง Record ใน Table B เท่านั้น เช่น ผู้ใช้หนึ่งคนมีโปรไฟล์หนึ่งชุด
  • 1:N (One-to-Many) — หนึ่ง Record ใน Table A สัมพันธ์กับหลาย Record ใน Table B เช่น ผู้ใช้หนึ่งคนมีหลายคำสั่งซื้อ (คุณสัมพันธ์ที่พบบ่อยที่สุด)
  • N:M (Many-to-Many) — หลาย Record ใน Table A สัมพันธ์กับหลาย Record ใน Table B เช่น นักศึกษาหนึ่งคนลงทะเบียนหลายวิชา และหนึ่งวิชามีหลายคนลงทะเบียน ต้องแก้ด้วยตาราง Junction (ตารางกลาง)
ตัวอย่าง ER Diagram: ระบบลงทะเบียนนักศึกษา

┌──────────────────┐       ┌──────────────────────┐       ┌────────────────┐
│    students      │       │   enrollments (N:M)  │       │    courses     │
├──────────────────┤       ├──────────────────────┤       ├────────────────┤
│ PK id            │ 1   N │ PK id                │ N   1 │ PK id          │
│    student_code  │──────►│ FK student_id        │◄──────│    code        │
│    first_name    │       │ FK course_id         │       │    name        │
│    last_name     │       │    enrolled_date      │       │    credits     │
│    email         │       │    grade             │       │    FK dept_id  │
│    faculty       │       └──────────────────────┘       └────────────────┘
│    created_at    │
└──────────────────┘

  students (1) ────── (N) enrollments (N) ────── (1) courses

MySQL ผ่าน phpMyAdmin

phpMyAdmin คือ Web Interface สำหรับจัดการ MySQL ที่มาพร้อมกับ XAMPP เข้าถึงได้ที่ http://localhost/phpmyadmin โดยไม่ต้องใช้รหัสผ่านเริ่มต้น

การดำเนินการหลักผ่าน phpMyAdmin:

  • สร้างฐานข้อมูล: คลิก "New" ที่แถบซ้าย → ตั้งชื่อ → เลือก Collation utf8mb4_unicode_ci → Create
  • สร้างตาราง: เลือก Database → "Create table" → ตั้งชื่อตาราง → กำหนดจำนวนคอลัมน์ → Go
  • กำหนดโครงสร้าง: ตั้งชื่อคอลัมน์, เลือก Type, กำหนด Length, Null/Not Null, Default, Index
  • เพิ่มข้อมูล: แท็บ "Insert" → กรอกข้อมูล → Go
  • ดูข้อมูล: แท็บ "Browse" แสดง Records ทั้งหมด
  • รัน SQL: แท็บ "SQL" สำหรับเขียนคำสั่ง SQL โดยตรง
เคล็ดลับ — Collation

ตั้ง Collation เป็น utf8mb4_unicode_ci เสมอเมื่อสร้าง Database หรือ Table utf8mb4 รองรับ Emoji และอักขระพิเศษ, unicode_ci ทำให้ค้นหาแบบ Case-Insensitive และรองรับภาษาไทยได้อย่างถูกต้อง

SQL พื้นฐาน

SQL (Structured Query Language) คือภาษามาตรฐานสำหรับจัดการฐานข้อมูล Relational แบ่งเป็น 4 กลุ่มหลัก (CRUD):

  • DDL (Data Definition Language): CREATE, ALTER, DROP — กำหนดโครงสร้าง
  • DML (Data Manipulation Language): INSERT, UPDATE, DELETE — จัดการข้อมูล
  • DQL (Data Query Language): SELECT — ดึงข้อมูล
  • DCL (Data Control Language): GRANT, REVOKE — จัดการสิทธิ์

คำสั่ง SELECT ที่ใช้บ่อย:

  • SELECT * FROM users — ดึงทุก Record และทุก Column
  • SELECT name, email FROM users WHERE id = 5 — เลือก Column ที่ต้องการ + กรอง
  • SELECT * FROM products WHERE price > 500 ORDER BY price DESC LIMIT 10
  • SELECT COUNT(*) AS total FROM users — นับจำนวน Record
  • SELECT * FROM users WHERE email LIKE '%@gmail.com' — ค้นหาแบบ Pattern

เชื่อมต่อ MySQL กับ PHP

PHP มี 2 วิธีหลักสำหรับเชื่อมต่อ MySQL:

  • MySQLi (MySQL Improved) — API แบบ Procedural และ Object-Oriented ใช้งานง่าย แต่ผูกกับ MySQL เท่านั้น (mysqli_connect())
  • PDO (PHP Data Objects) — Object-Oriented Interface รองรับหลาย Database (MySQL, PostgreSQL, SQLite) ด้วย Prepared Statements ที่ป้องกัน SQL Injection ได้ดีกว่า แนะนำให้ใช้ PDO 
การเชื่อมต่อ PHP → MySQL ด้วย PDO:

  PHP Script
      │
      ├─ new PDO("mysql:host=localhost;dbname=mydb;charset=utf8mb4", $user, $pass)
      │
      ▼
  PDO Object
      │
      ├─ $stmt = $pdo->prepare("SELECT * FROM users WHERE id = ?");
      ├─ $stmt->execute([$id]);  ← Prepared Statement ป้องกัน SQL Injection
      └─ $rows = $stmt->fetchAll(PDO::FETCH_ASSOC);
      │
      ▼
  Array ของข้อมูล → แสดงใน HTML
ข้อควรระวัง — SQL Injection

ห้ามนำตัวแปรจากผู้ใช้ใส่ใน SQL Query โดยตรง เช่น "SELECT * FROM users WHERE id = " . $_GET['id'] ผู้ใช้อาจส่ง 1 OR 1=1 ซึ่งจะ Return ทุก Record ในตาราง หรือ 1; DROP TABLE users; ป้องกันด้วยการใช้ Prepared Statements ของ PDO หรือ MySQLi เสมอ

Prepared Statements ทำงานโดยแยกโครงสร้าง SQL ออกจากข้อมูล: ? หรือ :name เป็น Placeholder ที่ PDO จะ Bind ข้อมูลให้อัตโนมัติ พร้อม Escape อักขระอันตรายทั้งหมด ทำให้ SQL Injection เป็นไปไม่ได้

create_tables.sql SQL 

-- ============================
-- สร้างฐานข้อมูลและตาราง
-- ============================

-- สร้าง Database
CREATE DATABASE IF NOT EXISTS mywebapp_db
  CHARACTER SET utf8mb4
  COLLATE utf8mb4_unicode_ci;

-- เลือกใช้ Database นี้
USE mywebapp_db;

-- ============================
-- ตาราง users: ข้อมูลผู้ใช้งาน
-- ============================
CREATE TABLE IF NOT EXISTS users (
  id          INT UNSIGNED     NOT NULL AUTO_INCREMENT,
  username    VARCHAR(50)      NOT NULL,
  email       VARCHAR(100)     NOT NULL,
  password    VARCHAR(255)     NOT NULL,  -- เก็บ Hashed password (bcrypt)
  first_name  VARCHAR(50)      NOT NULL,
  last_name   VARCHAR(50)      NOT NULL,
  role        ENUM('user','admin','moderator') NOT NULL DEFAULT 'user',
  is_active   TINYINT(1)       NOT NULL DEFAULT 1,
  created_at  DATETIME         NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at  DATETIME         NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,

  PRIMARY KEY (id),
  UNIQUE KEY uk_username (username),
  UNIQUE KEY uk_email    (email),
  KEY idx_role (role)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

-- ============================
-- ตาราง categories: หมวดหมู่สินค้า
-- ============================
CREATE TABLE IF NOT EXISTS categories (
  id          INT UNSIGNED  NOT NULL AUTO_INCREMENT,
  name        VARCHAR(100)  NOT NULL,
  slug        VARCHAR(100)  NOT NULL,
  description TEXT,
  created_at  DATETIME      NOT NULL DEFAULT CURRENT_TIMESTAMP,

  PRIMARY KEY (id),
  UNIQUE KEY uk_slug (slug)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

-- ============================
-- ตาราง products: สินค้า
-- ============================
CREATE TABLE IF NOT EXISTS products (
  id           INT UNSIGNED    NOT NULL AUTO_INCREMENT,
  name         VARCHAR(200)    NOT NULL,
  description  TEXT,
  price        DECIMAL(10, 2)  NOT NULL DEFAULT 0.00,
  stock        INT             NOT NULL DEFAULT 0,
  image_name   VARCHAR(255),
  category_id  INT UNSIGNED,              -- Foreign Key ไปยัง categories
  is_active    TINYINT(1)      NOT NULL DEFAULT 1,
  created_at   DATETIME        NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at   DATETIME        NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,

  PRIMARY KEY (id),
  KEY idx_category (category_id),
  KEY idx_price    (price),
  CONSTRAINT fk_product_category
    FOREIGN KEY (category_id) REFERENCES categories (id)
    ON DELETE SET NULL
    ON UPDATE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

-- ============================
-- ตาราง orders: คำสั่งซื้อ (1:N กับ users)
-- ============================
CREATE TABLE IF NOT EXISTS orders (
  id           INT UNSIGNED   NOT NULL AUTO_INCREMENT,
  user_id      INT UNSIGNED   NOT NULL,
  total_price  DECIMAL(12, 2) NOT NULL DEFAULT 0.00,
  status       ENUM('pending','paid','shipped','completed','cancelled') NOT NULL DEFAULT 'pending',
  note         TEXT,
  created_at   DATETIME       NOT NULL DEFAULT CURRENT_TIMESTAMP,

  PRIMARY KEY (id),
  KEY idx_user   (user_id),
  KEY idx_status (status),
  CONSTRAINT fk_order_user
    FOREIGN KEY (user_id) REFERENCES users (id)
    ON DELETE RESTRICT
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

-- ============================
-- SELECT: ดึงข้อมูล
-- ============================

-- ดึงทุก Record
SELECT * FROM users;
SELECT * FROM products;

-- เลือก Column ที่ต้องการ
SELECT id, username, email, created_at FROM users;

-- กรองด้วย WHERE
SELECT * FROM products WHERE price > 1000;
SELECT * FROM users WHERE email LIKE '%@gmail.com';
SELECT * FROM products WHERE stock = 0;

-- เรียงลำดับ
SELECT * FROM products ORDER BY price DESC;
SELECT * FROM products ORDER BY price ASC LIMIT 3;

-- JOIN: รวมข้อมูลจากหลายตาราง
SELECT
  p.id,
  p.name          AS product_name,
  p.price,
  c.name          AS category_name
FROM products p
INNER JOIN categories c ON p.category_id = c.id
ORDER BY c.name, p.price;

-- Aggregate Functions
SELECT COUNT(*) AS total_products FROM products;
SELECT AVG(price) AS avg_price FROM products;
SELECT MIN(price) AS cheapest, MAX(price) AS most_expensive FROM products;
SELECT category_id, COUNT(*) AS product_count, AVG(price) AS avg_price
FROM products
GROUP BY category_id
ORDER BY product_count DESC;

-- ============================
-- UPDATE: แก้ไขข้อมูล
-- ============================
UPDATE products SET price = 24900.00, stock = 4 WHERE id = 1;
UPDATE users SET is_active = 0 WHERE id = 3; -- ระงับบัญชี

-- ============================
-- DELETE: ลบข้อมูล
-- ============================
DELETE FROM products WHERE stock = 0 AND is_active = 0;
-- ระวัง! ไม่มี WHERE จะลบทุก Record

  • ถ่ายภาพหน้าจอโครงสร้างทั้งสองตาราง
เกณฑ์การผ่าน

ใน phpMyAdmin แสดงฐานข้อมูล mywebapp_db พร้อมตาราง users และ products เมื่อคลิก "Structure" ของแต่ละตารางต้องเห็น Columns ครบตามที่กำหนด Primary Key และ Constraints ถูกต้อง

คำแนะนำ

เมื่อสร้างตาราง ให้คลิกที่ช่อง "Index" ของ id และเลือก "PRIMARY" พร้อมติ๊ก "AUTO_INCREMENT" สำหรับ username และ email ให้เลือก "UNIQUE" ใน Index เพื่อป้องกันค่าซ้ำ

2
เขียน SQL สร้างตาราง

เขียนคำสั่ง SQL สำหรับสร้างตาราง users และ products ด้วยตนเองใน phpMyAdmin SQL tab

  • ใน phpMyAdmin คลิก Database mywebapp_db → แท็บ "SQL"
  • เขียนคำสั่ง DROP TABLE IF EXISTS products, users; เพื่อลบตารางเดิมก่อน (ถ้ามี)
  • เขียนคำสั่ง CREATE TABLE users (...) ด้วย Constraints ครบถ้วน: NOT NULL, UNIQUE, DEFAULT, PRIMARY KEY
  • เขียนคำสั่ง CREATE TABLE products (...) พร้อม FOREIGN KEY อ้างอิง users.id (ถ้าต้องการ)
  • เพิ่มคอลัมน์ updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP ในทั้งสองตาราง
เกณฑ์การผ่าน

รัน SQL แล้วไม่มี Error, ดูโครงสร้างตารางใน phpMyAdmin และ Export SQL ออกมาเก็บไว้ SQL ที่เขียนต้องมี Comment อธิบายแต่ละ Column ว่าทำหน้าที่อะไร

คำแนะนำ

ลำดับสำคัญ: ถ้า products มี FOREIGN KEY ไปยัง users ต้องสร้าง users ก่อน และ DROP products ก่อน DROP users ใช้ SET FOREIGN_KEY_CHECKS = 0; ก่อน DROP เพื่อปิดการตรวจสอบชั่วคราว

3
เพิ่มข้อมูลทดสอบ

เขียนคำสั่ง INSERT เพื่อเพิ่มข้อมูลทดสอบในทั้งสองตาราง

  • INSERT ผู้ใช้อย่างน้อย 5 คน ด้วย username, email, และ password (ใช้ค่า placeholder เช่น 'hashed_password')
  • INSERT สินค้าอย่างน้อย 5 ชิ้น ด้วยชื่อ ราคา และจำนวน Stock ที่หลากหลาย
  • ทำให้ข้อมูลสมจริง: ราคาสินค้าบางชิ้นเกิน 1000 บาท บางชิ้นน้อยกว่า
  • สินค้าบางชิ้น Stock = 0 (สินค้าหมด)
  • ใช้ INSERT INTO table (col1, col2) VALUES (v1, v2), (v1, v2), ... แบบ Multi-row
เกณฑ์การผ่าน

แท็บ Browse ของ users แสดงอย่างน้อย 5 Records และ products แสดงอย่างน้อย 5 Records ข้อมูลแต่ละ Record ครบทุก Field ที่กำหนด NOT NULL

คำแนะนำ

ถ้า INSERT แล้วได้ Error "Duplicate entry" แสดงว่าค่า username หรือ email ซ้ำ (เพราะตั้ง UNIQUE) ให้ใช้ค่าที่ไม่ซ้ำกัน หรือ TRUNCATE TABLE users; เพื่อลบข้อมูลเก่าทั้งหมดก่อน

4
ฝึก SQL SELECT

เขียนคำสั่ง SELECT หลายรูปแบบเพื่อดึงข้อมูลตามเงื่อนไขต่าง ๆ

  • ดึงผู้ใช้ทั้งหมด: SELECT * FROM users;
  • ดึงผู้ใช้ที่ email มี @gmail: SELECT * FROM users WHERE email LIKE '%@gmail.com';
  • ดึงสินค้าที่ราคา > 100 เรียงจากแพงสุด: SELECT * FROM products WHERE price > 100 ORDER BY price DESC;
  • ดึงสินค้าที่ Stock = 0 (หมด): SELECT * FROM products WHERE stock = 0;
  • นับจำนวนสินค้าทั้งหมดและราคาเฉลี่ย: SELECT COUNT(*) AS total, AVG(price) AS avg_price FROM products;
  • ดึงเฉพาะ username และ email จาก users 3 คนแรก: SELECT username, email FROM users LIMIT 3;
เกณฑ์การผ่าน

รันคำสั่ง SELECT ทุกข้อใน phpMyAdmin SQL tab และได้ผลลัพธ์ที่ถูกต้อง บันทึก Screenshot ผลลัพธ์แต่ละคำสั่งพร้อม SQL ที่ใช้

คำแนะนำ

ใน phpMyAdmin สามารถคลิกชื่อ Column ในผลลัพธ์เพื่อเรียงลำดับได้เลย หรือใช้แท็บ "Browse" แล้วคลิก Filter icon เพื่อกรองข้อมูลโดยไม่ต้องเขียน SQL

5
เชื่อมต่อ PHP กับ MySQL และแสดงผล

สร้างไฟล์ PHP ที่เชื่อมต่อ MySQL ผ่าน PDO และดึงข้อมูลมาแสดงเป็นตาราง HTML

  • สร้าง htdocs/mywebapp/config/db.php ที่มีโค้ด PDO Connection
  • สร้าง htdocs/mywebapp/users.php ที่ require_once ไฟล์ config แล้วรัน SELECT query
  • แสดงข้อมูล users ทั้งหมดเป็น HTML Table พร้อม Header: #, Username, Email, วันที่สมัคร
  • แสดงจำนวน Record ทั้งหมดที่ดึงมา
  • ใช้ htmlspecialchars() กับทุกค่าที่แสดงใน HTML
  • เพิ่ม Styling CSS ให้ตารางสวยงาม
เกณฑ์การผ่าน

เปิด http://localhost/mywebapp/users.php แล้วเห็นตารางที่แสดงข้อมูลจาก Database จริง ๆ ข้อมูลตรงกับที่ INSERT ไว้ในตาราง users และไม่มี PHP Error หรือ PDO Exception แสดงในหน้า

คำแนะนำ

หาก PDO ต่อไม่ได้ ให้ตรวจสอบ: (1) MySQL ใน XAMPP Control Panel เปิดอยู่หรือไม่, (2) ชื่อ Database ถูกต้องหรือไม่ (case-sensitive บน Linux), (3) Password ถูกต้องหรือไม่ (XAMPP default: user=root, password=ว่าง) เปิด Error Reporting ด้วย ini_set('display_errors', 1); ที่บรรทัดแรกชั่วคราว
