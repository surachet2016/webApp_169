1. การอัปโหลดไฟล์ใน PHP

เมื่อผู้ใช้อัปโหลดไฟล์ผ่านฟอร์ม PHP จะเก็บข้อมูลไฟล์ทั้งหมดใน Superglobal $_FILES ซึ่งมีโครงสร้างแบบ Array หลายมิติ โดยมี Key ที่สำคัญดังนี้: $_FILES['field']['name'] คือชื่อไฟล์ต้นฉบับ, $_FILES['field']['type'] คือ MIME Type ที่ Browser ส่งมา, $_FILES['field']['tmp_name'] คือ Path ของไฟล์ที่เก็บไว้ชั่วคราวบนเซิร์ฟเวอร์, $_FILES['field']['error'] คือ Error Code และ $_FILES['field']['size'] คือขนาดไฟล์ในหน่วย bytes

สิ่งที่ขาดไม่ได้สำหรับ HTML Form ที่ต้องการรับไฟล์คือ Attribute enctype="multipart/form-data" บน tag <form> ถ้าลืมใส่ $_FILES จะว่างเปล่าและอัปโหลดไม่ได้เลย นอกจากนี้ Method ต้องเป็น POST เท่านั้น เพราะ GET ไม่รองรับการส่งข้อมูลไฟล์

กระบวนการอัปโหลดไฟล์มี 4 ขั้นตอนหลัก: (1) ผู้ใช้เลือกไฟล์จาก Browser, (2) Browser ส่งไฟล์ไปยัง Server ผ่าน HTTP POST, (3) PHP เก็บไฟล์ไว้ที่ Temporary Directory (โดยปกติคือ /tmp), (4) Script PHP ย้ายไฟล์ไปยังตำแหน่งถาวรด้วย move_uploaded_file()

2. การตรวจสอบไฟล์ที่อัปโหลด

การตรวจสอบไฟล์ที่ถูกต้องเป็นขั้นตอนที่ขาดไม่ได้ก่อนบันทึกไฟล์ ต้องตรวจสอบอย่างน้อย 4 อย่าง: (1) Error Code — ค่า $_FILES['file']['error'] ต้องเป็น UPLOAD_ERR_OK (ค่า 0) ถึงจะอัปโหลดสำเร็จ, (2) ขนาดไฟล์ — ตรวจสอบ size ว่าไม่เกินขีดจำกัดที่กำหนด เช่น 2 MB = 2 * 1024 * 1024 bytes, (3) MIME Type — ใช้ mime_content_type() หรือ getimagesize() ตรวจสอบประเภทไฟล์จาก Content จริง ไม่ใช่จาก $_FILES['type'] ที่ Browser ส่งมา, (4) นามสกุลไฟล์ — ตรวจสอบ Extension ว่าอยู่ใน Whitelist

เหตุผลที่ห้ามเชื่อ $_FILES['type'] จาก Browser เพราะ Client สามารถปลอมแปลงค่านี้ได้ง่ายมาก เช่น ผู้โจมตีอาจส่งไฟล์ .php แต่ระบุ type เป็น "image/jpeg" เพื่อหลอกอัปโหลดขึ้นไปบนเซิร์ฟเวอร์แล้วเรียกใช้งาน ดังนั้นต้องตรวจสอบจากฝั่ง Server เสมอ

ข้อควรระวัง

อย่าให้ผู้ใช้อัปโหลดไฟล์ PHP หรือไฟล์ปฏิบัติการ (.php, .sh, .exe) — อาจทำให้แฮ็กเกอร์รันโค้ดบนเซิร์ฟเวอร์ได้! ควรใช้ Whitelist ของประเภทไฟล์ที่อนุญาตเท่านั้น

3. การย้ายและจัดเก็บไฟล์

ฟังก์ชัน move_uploaded_file($tmp_name, $destination) ใช้สำหรับย้ายไฟล์จาก Temporary Directory ไปยังตำแหน่งถาวร ฟังก์ชันนี้ปลอดภัยกว่า rename() เพราะตรวจสอบว่าไฟล์มาจากการอัปโหลดจริง ไม่ใช่ไฟล์ที่สร้างขึ้นใน Server

ปัญหาที่พบบ่อยคือชื่อไฟล์ชนกัน เช่น ผู้ใช้ 2 คนอัปโหลดไฟล์ชื่อ "photo.jpg" พร้อมกัน วิธีแก้คือสร้างชื่อไฟล์ที่ไม่ซ้ำกัน เช่น ใช้ uniqid(), time(), md5(), หรือ random_bytes() ผสมกับ Extension เดิม เช่น uniqid('img_', true) . '.jpg'

ควรจัดโครงสร้าง Folder อัปโหลดให้เป็นระเบียบ เช่น แบ่งตามปี/เดือน (uploads/2025/06/) เพื่อป้องกันไม่ให้ไฟล์กองรวมอยู่ใน Folder เดียวจนมีจำนวนมากเกินไป ซึ่งอาจทำให้ระบบไฟล์ช้าลง และต้องตรวจสอบ Permission ของ Folder ด้วย chmod 755 หรือ chmod 775

เคล็ดลับ

ตั้งชื่อไฟล์ใหม่เสมอแทนการใช้ชื่อเดิมจากผู้ใช้ เพราะชื่อไฟล์ต้นฉบับอาจมีอักขระพิเศษหรือ Path Traversal เช่น ../../etc/passwd ที่เป็นอันตราย

4. ความปลอดภัยของการอัปโหลด

การอัปโหลดไฟล์เป็นหนึ่งในช่องโหว่ที่แฮ็กเกอร์มักโจมตีมากที่สุด เพราะหากอนุญาตให้อัปโหลดไฟล์ PHP ขึ้นไปบน Web Root และเรียกผ่าน URL ได้ แฮ็กเกอร์จะสามารถรันคำสั่งใด ๆ บนเซิร์ฟเวอร์ได้ทันที (Remote Code Execution) มาตรการป้องกันสำคัญได้แก่:

1. ตรวจสอบ MIME Type จากเซิร์ฟเวอร์ ด้วย mime_content_type() หรือ getimagesize() ซึ่งอ่านจาก Magic Bytes ของไฟล์จริง ไม่ใช่ Header ที่ Browser ส่งมา 2. เก็บไฟล์นอก Web Root เช่นที่ /var/uploads/ แทน /var/www/html/uploads/ เพื่อป้องกันการเรียกไฟล์ผ่าน URL โดยตรง แล้วให้ PHP เป็นตัวส่งไฟล์ให้ผ่าน readfile() 3. สร้าง .htaccess ใน uploads/ เพื่อบล็อก PHP Execution

หมายเหตุ — .htaccess บล็อก PHP ใน uploads/

สร้างไฟล์ uploads/.htaccess ที่มีเนื้อหา: php_flag engine off เพื่อป้องกันไม่ให้ไฟล์ PHP ใน uploads/ ถูก Execute แม้ว่าแฮ็กเกอร์จะอัปโหลดขึ้นไปได้

function validateImageUpload(array $file, int $maxSize = 2097152): array
{
    // ตรวจสอบ Upload Error
    if ($file['error'] !== UPLOAD_ERR_OK) {
        $messages = [
            UPLOAD_ERR_INI_SIZE   => 'ไฟล์ใหญ่เกินค่าที่กำหนดใน php.ini',
            UPLOAD_ERR_FORM_SIZE  => 'ไฟล์ใหญ่เกินค่าที่กำหนดในฟอร์ม',
            UPLOAD_ERR_PARTIAL    => 'ไฟล์อัปโหลดไม่สมบูรณ์',
            UPLOAD_ERR_NO_FILE    => 'ไม่มีไฟล์ถูกอัปโหลด',
            UPLOAD_ERR_NO_TMP_DIR => 'ไม่พบโฟลเดอร์ชั่วคราว',
            UPLOAD_ERR_CANT_WRITE => 'ไม่สามารถเขียนไฟล์ลง Disk ได้',
        ];
        return ['ok' => false, 'error' => $messages[$file['error']] ?? 'ข้อผิดพลาดไม่ทราบสาเหตุ'];
    }

    // ตรวจสอบขนาด
    if ($file['size'] > $maxSize) {
        $mb = number_format($maxSize / 1048576, 0);
        return ['ok' => false, 'error' => "ไฟล์ต้องไม่เกิน {$mb} MB"];
    }

    // ตรวจสอบว่าเป็นรูปภาพจริงด้วย getimagesize()
    $imgInfo = @getimagesize($file['tmp_name']);
    if ($imgInfo === false) {
        return ['ok' => false, 'error' => 'ไฟล์ที่อัปโหลดไม่ใช่รูปภาพ'];
    }

    // ตรวจสอบ MIME จาก Magic Bytes
    $allowedMime = ['image/jpeg', 'image/png', 'image/gif', 'image/webp'];
    $mime = mime_content_type($file['tmp_name']);
    if (!in_array($mime, $allowedMime)) {
        return ['ok' => false, 'error' => "MIME Type ไม่อนุญาต: {$mime}"];
    }

    // ตรวจสอบ Extension
    $allowedExt = ['jpg', 'jpeg', 'png', 'gif', 'webp'];
    $ext = strtolower(pathinfo($file['name'], PATHINFO_EXTENSION));
    if (!in_array($ext, $allowedExt)) {
        return ['ok' => false, 'error' => "นามสกุลไฟล์ไม่อนุญาต: .{$ext}"];
    }

    return [
        'ok'        => true,
        'error'     => '',
        'extension' => $ext,
        'mime'      => $mime,
        'width'     => $imgInfo[0],
        'height'    => $imgInfo[1],
    ];
}

// ตัวอย่างการใช้งาน
if (isset($_FILES['photo'])) {
    $result = validateImageUpload($_FILES['photo'], 2 * 1024 * 1024);
    if ($result['ok']) {
        $newName = uniqid('photo_') . '.' . $result['extension'];
        move_uploaded_file($_FILES['photo']['tmp_name'], 'uploads/' . $newName);
        echo "อัปโหลดสำเร็จ: {$newName} ({$result['width']}x{$result['height']} px)";
    } else {
        echo "ข้อผิดพลาด: " . htmlspecialchars($result['error']);
    }
}

🧪 ปฏิบัติการ Lab 6 — อัปโหลดไฟล์

สร้างระบบอัปโหลดและจัดการรูปภาพสินค้า ตั้งแต่การรับไฟล์ การตรวจสอบความปลอดภัย จนถึงการเชื่อมโยงกับฐานข้อมูล

1
ติดตั้ง uploads/ folder

สร้างโฟลเดอร์ uploads/ ในโปรเจกต์ และทดสอบว่า PHP สามารถเขียนไฟล์เข้าไปได้ สร้างไฟล์ uploads/.htaccess เพื่อป้องกันการ Execute PHP

ทดสอบการเขียนไฟล์ PHP
เกณฑ์การผ่าน

รัน test_upload_dir.php แล้วเห็น "โฟลเดอร์ uploads/ พร้อมใช้งาน" และ "สร้าง .htaccess สำเร็จ" / โฟลเดอร์ uploads/ มีไฟล์ .htaccess อยู่

คำแนะนำ

บน Windows (XAMPP) โฟลเดอร์ปกติมักเขียนได้เลย บน Linux อาจต้องรัน chmod 755 uploads/ หรือ chown www-data:www-data uploads/

2
อัปโหลดรูปภาพพร้อม Validation

สร้างหน้า upload.php ที่มีฟอร์มรับไฟล์รูปภาพ ต้องตรวจสอบ: เฉพาะไฟล์รูปภาพ (jpg, png, gif, webp) เท่านั้น, ขนาดไม่เกิน 2 MB, ตั้งชื่อใหม่ด้วย uniqid() ก่อนบันทึก

แสดง Error Message ที่ชัดเจนเมื่อไฟล์ไม่ผ่าน เช่น "ไฟล์ใหญ่เกิน 2 MB" หรือ "ประเภทไฟล์ไม่ถูกต้อง" และแสดงชื่อไฟล์ที่บันทึกแล้วเมื่อสำเร็จ

เกณฑ์การผ่าน

อัปโหลด jpg ปกติ → แสดงข้อความสำเร็จ / อัปโหลดไฟล์ .pdf → แสดง error / อัปโหลดไฟล์เกิน 2 MB → แสดง error / ชื่อไฟล์ใน uploads/ ไม่ใช่ชื่อต้นฉบับจากผู้ใช้

คำแนะนำ

ใช้ mime_content_type($file['tmp_name']) แทน $file['type'] เพื่อความปลอดภัย และตรวจสอบ error code ก่อน (UPLOAD_ERR_OK === 0) ก่อนทำขั้นตอนอื่น

3
แสดงรูปภาพที่อัปโหลดเป็น Gallery

สร้างหน้า gallery.php ที่ใช้ glob() ดึงรายการไฟล์รูปภาพทั้งหมดใน uploads/ แล้วแสดงเป็น Grid Gallery ที่มีรูปภาพ ชื่อไฟล์ และขนาดไฟล์

เรียงรายการจากใหม่ไปเก่า (usort + filemtime) และแสดงจำนวนรูปทั้งหมดที่ Header

เกณฑ์การผ่าน

หลังอัปโหลดรูปใน Task 2 เปิด gallery.php → เห็นรูปภาพแสดงเป็น Grid / แสดงชื่อไฟล์และขนาดใต้รูป / รูปล่าสุดแสดงก่อน

คำแนะนำ

ใช้ glob('uploads/*.{jpg,jpeg,png,gif,webp}', GLOB_BRACE) เพื่อดึงทุก Extension ในครั้งเดียว และ loading="lazy" บน img tag เพื่อประสิทธิภาพ

4
เชื่อมรูปภาพกับฐานข้อมูล (Products)

เพิ่ม Column image_name VARCHAR(255) ในตาราง products จากนั้นแก้ไข create.php ให้มีช่องอัปโหลดรูปสินค้าด้วย บันทึกชื่อไฟล์ (ไม่ใช่ Path เต็ม) ลงใน Column image_name

แก้ไข list.php ให้แสดงรูปสินค้าในตาราง โดยถ้า image_name ว่างให้แสดงรูป Placeholder แทน

เกณฑ์การผ่าน

เพิ่มสินค้าพร้อมรูป → list.php แสดงรูปสินค้าถูกต้อง / เพิ่มสินค้าไม่มีรูป → list.php แสดง Placeholder แทน / ตาราง products มี Column image_name

คำแนะนำ

เพิ่ม Column ด้วย SQL: ALTER TABLE products ADD COLUMN image_name VARCHAR(255) DEFAULT NULL; และบันทึกเฉพาะชื่อไฟล์ เช่น "img_abc123.jpg" ไม่ใช่ Path เต็ม เพื่อให้ยืดหยุ่นเมื่อย้าย Server

5
ลบไฟล์พร้อมกับการลบสินค้า

แก้ไข delete.php ให้เมื่อลบสินค้า ให้ตรวจสอบว่าสินค้านั้นมี image_name หรือไม่ ถ้ามีให้ลบไฟล์รูปออกจาก uploads/ ด้วย unlink() พร้อมกัน

ต้องตรวจสอบว่าไฟล์มีอยู่จริงด้วย file_exists() ก่อน unlink เพื่อป้องกัน Warning เมื่อไฟล์ถูกลบไปก่อนหน้าแล้ว

เกณฑ์การผ่าน

ลบสินค้าที่มีรูป → รูปหายออกจาก uploads/ ด้วย / ลบสินค้าที่ไม่มีรูป → ไม่เกิด Error / ตรวจสอบด้วย gallery.php ว่ารูปหายจริง

คำแนะนำ

ใช้ if ($product['image_name'] && file_exists($path)) { unlink($path); } ลบ Record จากฐานข้อมูลก่อน แล้วค่อยลบไฟล์ เพื่อป้องกัน Orphan files กรณีที่ DELETE SQL ล้มเหลว
